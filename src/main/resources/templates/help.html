AI 助理 v1.0 — 功能指南与常见问题

🚀 快速开始

AI 助理基于 Spring AI 2.0 + Spring Boot 4.1，直连 LLM API，无需外部编排服务。启动后访问 /config 配置 LLM API Key 和笔记库即可使用全部功能。

前置依赖

依赖说明必须
Java 17+运行时环境✅
LLM API KeyDeepSeek / 商汤 / 智谱等兼容 API✅
语义检索Ollama 本地 或 API（硅基流动等）🔘 可选

启动步骤

# 1. 编译
mvn package -q
# 2. 启动
java -jar target/ai-assistant-1.0.0.jar
# 3. 打开浏览器访问 http://localhost:6790
# 4. 在 /config 页面配置 LLM API Key + 笔记库路径

提示：先配 API Key 和笔记库路径，否则 AI 功能和文件操作不可用。支持多 LLM 模型同时配置，可在对话页切换。

每个笔记库独立管理

  • 笔记库目录 — 各笔记库可配置不同的笔记存储路径
  • 笔记目录 — 笔记库根目录下的一级目录自动识别为笔记分区
  • 任务/提醒 — 任务看板和定时提醒按笔记库隔离
  • 聊天历史 — AI 对话记录按笔记库独立保存
  • AI 分析 — 日报生成、数据洞察等按笔记库独立运行

隔离机制

维度隔离方式
聊天历史sessions.kb_id + 向量召回限定 kb_id
笔记目录笔记库根目录自动扫描，支持排序/隐藏
AI 记忆{notesDir}/AI/记忆/ 隔离
任务/提醒{notesDir}/AI/任务/ + AI/提醒/ 隔离
文件操作NoteTools 路径基于当前笔记库

💬 AI 对话

连续对话模式，按当前笔记库加载聊天历史，SSE 流式输出。AI 可自主调用工具操作笔记库。

工作模式

  • 笔记库对话（默认）— 注入历史上下文 + 语义检索，适合基于笔记库的问答
  • 创作式对话 — 无历史上下文，适合独立创作或开放讨论

SSE 实时流式响应

事件说明
session建立会话，返回 sessionId，前端更新 URL
status进度指示（处理中/读取文件/写入等）
textAI 回复文本
done流式完成
error错误信息

AI 工具（@Tool 自动编排）

AI 根据对话内容自主决定是否调用以下工具，无需手动指定：

工具功能触发场景
readFile读取笔记库文件内容AI 需要了解数据格式或历史内容
writeFile写入文件（先读后写，防覆盖）创建/更新记录、文档
listDir列出目录内容AI 需要探索笔记库结构
searchFiles按文件名关键词搜索查找特定文件

语义检索 / 长期记忆

  • Ollama 本地 — 不填 API Key，使用本地 Ollama（推荐 nomic-embed-text）
  • API 模式 — 填入 API Key，支持硅基流动等 OpenAI 兼容服务

AI 回复后自动生成向量并跨会话检索相似轮次（余弦相似度阈值 0.5），最多从 8 轮最相关的历史中提取上下文。不可用时回退最近 3 轮对话。

对话管理

  • 左侧历史会话列表，支持切换、删除
  • 消息自动保存至 SQLite，刷新页面不丢失
  • 支持将对话导出为 Markdown 保存至笔记库

📂 笔记库详情（/kb/{id}）

每个笔记库拥有独立的笔记目录浏览、AI 分析、任务提醒、AI 指南、日报等功能。

笔记目录

笔记库根目录下的一级目录自动识别为笔记分区，进入后可浏览文件、进行 AI 分析。

  • 目录列表 — 展示根目录下一级目录，支持拖拽排序和隐藏
  • 文件浏览 — 目录下的 Markdown 文件列表，在线预览
  • AI 分析 — SSE 流式分析目录内容并生成洞察报告，支持自定义提示词
  • 原始数据 — 目录下 data/ 中的 JSON 数据文件管理

笔记浏览

  • 目录树 — 浏览当前笔记库文件目录结构
  • Markdown 预览 — 渲染为 HTML 查看
  • 文件操作 — 新建 .md 文件、删除文件

AI 指南

管理当前笔记库的 AGENTS.md 规则文件和 AI 记忆文件。AGENTS.md 在 AI 对话中每次都会读取，定义笔记库目录结构和操作规范。

AI 综合日报

  • 自动生成：读取 AGENTS.md + AI 记忆 + 当日工作数据，AI 生成日报
  • 支持自定义分析提示词
  • 可选通过飞书 Webhook 推送

📋 计划管理

任务看板 + 定时提醒统一管理，数据按笔记库隔离。

任务看板

  • 任务字段：标题、描述、状态（待办/完成）、优先级（高/中/低）、截止日期
  • 按笔记库隔离，名称可自定义（如"学习计划"）
  • 数据存储在 {notesDir}/AI/任务/data.json

定时提醒

  • 支持重复类型：一次性、每日、每周、每月、每年
  • 按笔记库隔离，名称可自定义（如"复习提醒"）
  • 通过飞书 Webhook 推送提醒消息

🗂️ 数据中心

集中管理结构化数据集，支持多数据集、Schema 定义、批量导入。

功能

  • 创建数据集（定义 Schema：字段名、类型、描述、必填、默认值）
  • Excel 导入 — 自动检测列头、模糊匹配字段映射，支持 AI 智能归一化
  • JSON 导入 — 直接粘贴或上传 JSON 数据
  • URL 导入 — AI 自动访问 URL 并提取结构化数据
  • 记录浏览、搜索、删除，自动 MD5 去重
  • 可将数据集导出到笔记库目录

🖼️ 图片识别

基于多模态 Vision API 的图片分析功能，按笔记库隔离。

  • 自动扫描当前笔记库中的图片（PNG / JPG / GIF / WebP / BMP）
  • 缩略图浏览，支持选择 Vision 模型
  • 直接上传本地图片进行分析
  • 分析结果可保存到当前笔记库

📡 数据采集器

定时从外部数据源采集数据，AI 解析后写入数据中心或笔记库。

  • 定义采集任务（URL/来源 + Cron 表达式）
  • AI 解析采集到的原始数据并提取结构化 JSON
  • 可关联数据集自动导入
  • 采集日志与执行结果查看

💻 编程 AI

独立飞书机器人 + Pi CLI 代码自动排查，与笔记库系统独立。

  • 通过飞书机器人接收代码问题，AI 自动排查
  • 使用 Pi CLI 执行代码分析
  • 在配置页可选开启，独立于笔记库系统

✈️ 飞书集成

支持 Webhook 消息推送和 WebSocket 长连接双向通信。

Webhook 推送（出站）

  • 日报推送、定时提醒推送
  • 配置页填写 Webhook URL 即可

WebSocket 长连接（双向）

  • 通过飞书 OpenAPI SDK 建立长连接
  • 实时接收群内消息，AI 处理后自动回复
  • 支持单聊和群聊，去重机制防止重复处理

配置步骤

  1. 在飞书开放平台创建应用，获取 App ID / App Secret
  2. 在配置页填写凭据并选择目标群
  3. 开启"消息接收"开关

⚙️ 系统配置

所有配置通过 /config 页面管理，无需手动编辑文件。

🏗️ 技术架构

组件技术

❓ 常见问题

问题原因解决
AI 回复"API Key 未配置"未在 /config 填写 API Key配置页填写 LLM API Key
AI 回复为空LLM 超时或返回空检查网络，超时设为 600s
"未配置笔记库"未添加笔记库或未配置路径在笔记库管理页添加笔记库
语义检索不可用Ollama 未启动或向量模型未配置启动 Ollama 或配置 API 模式的向量模型
飞书消息收不到WebSocket 未连接或 Webhook 未配置在配置页检查飞书凭据
对话记录丢失切换了笔记库每个笔记库的对话是独立的，切换回对应笔记库即可
