📖 老齐AI个人助理

一站式个人工作台，集成 AI 综合日报、对话问答、笔记管理、公众号运营分析、客户管理等功能。 所有 AI 能力基于本地笔记库和 opencode serve，不依赖任何第三方云服务。

🚀 快速启动

环境要求

依赖版本用途
Java≥17Web 服务运行环境
opencodelatestAI 引擎（必须启动两个实例）

启动步骤

# 1. 先启动两个 opencode serve 服务（必须）
opencode serve --port 14096  # AI聊天/笔记库服务
opencode serve --port 14099  # 代码分析服务

# 2. 编译项目
mvn package -q

# 3. 启动应用
java -jar target/assistant-v2-2.0.0.jar

# 访问地址
# - Web UI: http://localhost:6790
# - 健康检查: http://localhost:6790/health

🧠 AI 引擎（opencode serve）

系统依赖两个 opencode serve 实例提供所有 AI 能力，缺一不可：

服务端口用途
AI聊天/笔记库14096综合日报、智能问答、数据采集、客户跟进分析
代码分析14099Java 项目分析、代码知识库

启动后可在顶部导航栏看到连接状态指示器。红色表示服务未启动，绿色表示正常。

⚙️ 首次使用配置

首次启动后，请先前往 配置页 完成以下设置：

  1. 笔记库根目录 — 设置笔记库的绝对路径（如 D:\projects\richie_learning_notes），所有数据文件的相对路径都基于此目录
  2. 聊天记录目录 — 默认 chat，对话记录保存在 {根目录}/chat/chat_sessions.json
  3. 综合日报目录 — 默认 工作/综合日报，AI 日报保存到此目录
  4. 数据目录 — 客户、运营、任务数据所在的子目录
  5. 飞书 Webhook — 可选，配置后可接收推送通知

未配置前，任何需要访问笔记库的操作会提示你去配置。

所有定时任务可在 配置页 手动触发执行。

📊 综合日报

首页展示 AI 综合日报，数据源为笔记库中的记忆文件，涵盖：项目进展、客户动态、今日 TODO、昨日回顾、学习方向、生活提醒、个人目标、AI 建议。 日报文件保存在 {笔记库根目录}/工作/综合日报/{日期}.md，可由定时任务自动生成，也支持 AI 对话手动触发。

💬 AI 对话

与 AI 助理对话，它会读取整个笔记库来回答问题。支持流式响应、实时状态显示、历史会话管理。

  • 流式响应 — 回答逐字显示，无需等待全部生成完毕
  • 实时状态 — AI 读取笔记时显示 📖 文件名，知道它在做什么
  • 历史会话 — 左侧边栏保存所有对话记录，可切换、可删除
  • 自动保存 — 每次对话自动保存到 chat_sessions.json

📝 快速数据录入

核心原则：规则由你在 AI 的 AGENTS.md 中制定，AI 自己保存数据。 Java 代码只做消息转发，不参与任何业务逻辑。

你只需在笔记库根目录的 AGENTS.md 中定义录入规则（字段映射、计算方式、保存格式等）， 所有入口的消息都会原样发给 AI，AI 按你的规则解析并直接写入数据文件。

入口方式说明
页面表单运营/客户/工作页面点按钮弹窗填写字段拼成文本发给 AI，AI 按规则保存
飞书消息以"记录/新增/添加/创建/录入"开头发消息原样转发 AI，AI 按规则保存并回复
AI对话聊天框以"记录/新增/添加/创建/录入"开头输入原样转发 AI，AI 按规则保存并回复

示例消息（AI 按 AGENTS.md 规则解析）

记录日报：今天修复了导出bug，下午跟进了张三的合同
记录文章：《实战指南》微信阅读3200点赞85，增粉12
新增客户：王五，公司启航科技，阶段意向，产品ERP
新增线索：赵六，来源转介绍，备注朋友推荐
记录跟进：张三 谈了合同细节，下周二再约
记录日统计：码农老齐 粉丝28500 阅读4200

📈 运营看板

公众号运营数据分析，数据自动读取 {笔记库根目录}/自媒体/data/ 目录下所有 JSON 文件， 包括文章数据、账号平台数据、每日流量统计。 提供各公众号粉丝走势、文章阅读量排名、平台对比、AI 自动分析内容策略效果、写作建议和选题推荐。

📂 数据编辑器

通用 JSON 数据编辑器，支持对所有已配置的数据目录进行增删改查操作。 在 配置页 设置数据目录后，编辑器会自动发现并展示目录下的所有 JSON 文件， 以表格形式展示数据，支持列设置、排序、搜索。

📩 飞书集成

支持两种飞书集成方式：

  • Webhook 推送 — 配置 Webhook URL 后，AI 综合日报、发文提醒、下班提醒等自动推送到飞书群
  • 消息轮询（API） — 配置自建应用凭据后，AI 助理可轮询指定群的未读消息并自动回复

均在 配置页 中设置。

👥 客户管理

客户管理看板展示概览、销售漏斗、合同金额、线索漏斗、近期活动、待跟进等数据。 数据存储在 {笔记库根目录}/企业/客户管理/ 目录，支持通过 AI 对话录入和更新。

📂 笔记管理

内置文件浏览器，无需 Obsidian 即可管理笔记库中的 Markdown 文件：

  • 浏览 — 按目录树逐层浏览所有文件
  • 查看 — Markdown 渲染为美观的 HTML
  • 复制 — 查看页有「📋 复制文章」按钮，可直接粘贴到公众号编辑器
  • 新建 — 在当前目录创建 Markdown 文件
  • 删除 — 删除 .md 文件

🗂️ 数据文件

所有数据以 JSON 文件形式存储在笔记库根目录下，无数据库依赖。 以下路径均为相对于笔记库根目录：

文件读写说明
{config-dir}/config.json读写应用配置（飞书、笔记库路径、数据目录等）
{config-dir}/assistant_log.json读写操作日志（最多保留 100 条）
chat/chat_sessions.json读写对话历史记录
工作/综合日报/读写AI 生成的综合日报文件（按日期命名）
自媒体/data/*.json读写公众号运营数据（文章、账号、日统计三个文件）
企业/客户管理/*.json读写客户、线索、跟进记录数据
记忆/当前重点.md只读TODO、近期动态、关键数据
记忆/项目状态.md只读各项目进展
AI记忆.md只读全局背景、产品矩阵、客户规模、战略决策
提醒.md读写每日循环和临时提醒（日报生成后自动清空临时提醒）
