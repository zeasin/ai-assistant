快速开始

AI 助理 v3.0 基于 Spring AI 2.0，直接调用 LLM API，无需外部编排服务。

前置依赖

依赖说明必须
Java 17+运行时✅
LLM API KeyDeepSeek 等兼容 API✅
Ollama语义检索加速🔘 可选

启动步骤

# 配置 API Key（/config 页面或环境变量）
# export DEEPSEEK_API_KEY=sk-xxxxx

# 启动应用
java -jar target/ai-assistant-3.0.0.jar

# 访问 http://localhost:6790

先去 /config 配置笔记库根目录和 API Key。

💬

对话模块

  • 知识库对话 — 结合笔记库历史和语义检索回答问题
  • 创作式对话 — 开启新会话，AI 无历史上下文
  • 笔记操作 — 让 AI 读/写笔记库文件
📊

业务数据

在笔记库中设置模块，每个模块可配置 JSON/CSV 数据文件，AI 自动分析给出洞察。

📸

图片识别

利用 LLM 多模态能力分析图片。支持 PNG / JPG / GIF / WebP / BMP。

☀️

日报系统

  • 工作日 09:00 — 自动生成综合日报
  • 工作日 18:00 — 下班提醒推送飞书
  • 日报存储在 AI/综合日报/
📡

数据采集

定时从指定 URL 采集数据，AI 分析提取为结构化 JSON，存入数据中心。

🔧

配置项

所有配置在 /config 页面管理。

配置项说明
笔记库根目录所有业务数据的存放根目录
API KeyLLM API 密钥
飞书集成Webhook URL 和应用凭证
🆘

常见问题

问题原因解决
AI 回复"API Key 未配置"未配置 API Key在 /config 填写
AI 回复为空超时或网络问题检查网络，超时设为 600s
语义检索不可用Ollama 未启动ollama run nomic-embed-text
🏗️

技术架构

  • Spring Boot 4.1 + Spring AI 2.0 + Java 17
  • AI 引擎：ChatClient 直连 LLM
  • 工具编排：@Tool + ToolCallingAdvisor
  • 语义检索：Ollama + nomic-embed-text
  • 存储：SQLite (MyBatis-Plus) + JSON
