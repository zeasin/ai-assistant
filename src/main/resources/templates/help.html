📖 AI 助理 (ai-assistant)

一站式个人工作台，集成 AI 综合日报、对话问答、笔记管理、数据采集等功能。 所有 AI 能力基于本地笔记库和 opencode serve，不依赖任何第三方云服务。

🚀 快速启动

环境要求

依赖版本用途
Java≥17Web 服务运行环境
opencodelatestAI 引擎（必须启动两个实例）

启动步骤

# 1. 启动 opencode serve 服务（必须）
opencode serve --port 14096

# 2. 编译项目
mvn package -q

# 3. 启动应用
java -jar target/ai-assistant-2.0.0.jar

# 访问地址
# - Web UI: http://localhost:6790
# - 健康检查: http://localhost:6790/health

⚙️ 首次使用配置

首次启动后，请先前往 配置页 完成以下设置：

  1. 笔记库根目录 — 设置笔记库的绝对路径（如 D:\notes），所有数据文件都基于此目录
  2. 飞书 Webhook — 可选，配置后可接收推送通知
  3. 字段标签 — 可选，自定义数据表格中英文 key 显示为中文标签

配置完成后，前往 AI 指南 设置 AGENTS.md 来定义 AI 的行为规则。

⏰ 定时任务

系统内置 Spring @Scheduled 驱动以下定时任务（时区：Asia/Shanghai）：

任务时间说明
综合日报生成每天 09:00AI 读取笔记库生成日报，推送飞书并保存到 AI/综合日报/

可在 配置页 手动触发执行。动态提醒可在 提醒页 配置。

📊 综合日报

首页展示 AI 综合日报，数据源为笔记库中的记忆文件，涵盖：TODO、学习方向、生活提醒、AI 建议。 日报文件保存在 {笔记库根目录}/AI/综合日报/{日期}.md，可由定时任务自动生成，也支持手动触发。

提示词可通过首页右上角 ⚙️ 按钮配置，存储在 AI/prompts/综合日报提示词.md。

💬 AI 对话

与 AI 助理对话，它会读取整个笔记库来回答问题。支持流式响应、实时状态显示、历史会话管理。

核心特性

  • 流式响应 — 回答逐字显示，无需等待全部生成完毕
  • 实时状态 — AI 读取笔记时显示 📖 文件名，知道它在做什么
  • 历史会话 — 左侧边栏保存所有对话记录，可切换、可删除

📦 模块系统

模块系统是管理各类数据的核心功能。每个模块包含：

  • 数据管理 — 以表格形式展示和编辑 JSON 数据
  • 文档浏览 — 浏览模块目录下的 Markdown 文件
  • AI 分析 — 基于模块数据进行 AI 分析，生成分析报告
  • 提示词配置 — 每个模块可独立配置 AI 分析的提示词

模块通过 /module/{id} 访问，配置在 配置页 中的 modules 字段定义。

🔧 数据采集器

数据采集器支持从外部数据源自动采集数据。每个采集任务包含：

  • 采集任务 — 定义采集目标和参数
  • 提示词模板 — AI 采集任务使用的提示词，存储在 AI/collector/prompts/
  • 定时执行 — 支持 cron 表达式定时采集
  • 手动触发 — 支持立即执行采集任务

📂 数据编辑器

通用 JSON 数据编辑器，支持对模块数据进行增删改查操作。 编辑器自动发现并展示数据文件，以表格形式展示，支持列设置、排序、搜索。

⏰ 定时提醒

定时提醒功能支持创建和管理各种循环提醒任务，到时间自动推送消息到飞书。

  • 每天 — 每天固定时间提醒，如下班日报提醒
  • 每周 — 每周特定星期几提醒
  • 每月 — 每月特定几号提醒
  • 每年 — 每年特定日期提醒
  • 一次 — 指定日期提醒一次

支持启用/禁用、编辑、删除等操作。提醒触发后会推送到配置的飞书群。 默认会创建"下班日报提醒"（每天 18:00）。

📂 笔记管理

内置文件浏览器，无需 Obsidian 即可管理笔记库中的 Markdown 文件：

  • 浏览 — 按目录树逐层浏览所有文件
  • 查看 — Markdown 渲染为美观的 HTML
  • 复制 — 查看页有「📋 复制文章」按钮，可直接粘贴到公众号编辑器
  • 新建 — 在当前目录创建 Markdown 文件
  • 删除 — 删除 .md 文件

🤖 AI 指南

管理 AI 的行为指南和记忆文件：

  • AGENTS.md — 定义 AI 的行为规则、数据录入规则等
  • AI 记忆 — AI 的长期记忆文件，存储在 AI/记忆/ 目录

📩 飞书集成

支持三种飞书集成方式，可根据需要组合使用。所有配置均在 配置页 中设置。

🔗 方式一：Webhook 推送（接收通知）

配置 Webhook URL 后，AI 综合日报、提醒等自动推送到飞书群。
仅需群机器人权限，配置最简单。

  1. 打开飞书，进入目标群聊的「设置」→「群机器人」→「添加机器人」
  2. 搜索并添加「自定义机器人（Webhook）」
  3. 复制机器人 Webhook 地址，格式：https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx
  4. 进入本系统 配置页，在「飞书 Webhook 配置」中粘贴地址并保存
  5. 点击「手动触发」区域的按钮测试推送是否正常

📡 方式二：消息轮询（API 方式接收消息）

通过飞书 API 定时轮询指定群的未读消息，AI 自动回复。
需创建飞书自建应用，配置复杂但功能完整。

第一步：创建飞书自建应用

  1. 打开 飞书开发者后台，点击「创建企业自建应用」
  2. 填写应用名称（如：AI 助理）、描述，创建完成后进入应用详情

第二步：获取 App ID 和 App Secret

  1. 在应用详情页左侧菜单点击「凭证与基础信息」
  2. 复制 App ID 和 App Secret（App Secret 首次需点击「显示」或重置）

第三步：配置权限

  1. 在左侧菜单点击「权限管理」
  2. 开启以下权限（必选）：
    • im:message — 获取单聊、群组消息
    • im:chat — 获取群列表信息
  3. 点击「批量开通」并确认

第四步：发布应用

  1. 在左侧菜单点击「版本管理与发布」
  2. 创建版本，填写版本号（如 1.0.0）、更新说明，点击「保存」
  3. 点击「申请发布」，选择审核方式
  4. 联系飞书管理员审批通过

第五步：将应用添加到群聊

  1. 打开要监听的飞书群，进入「设置」→「群机器人」→「添加机器人」
  2. 搜索并添加刚才创建的自建应用

第六步：在系统中配置

  1. 进入本系统 配置页，在「飞书消息接收（API 轮询）」区域
  2. 填入 App ID 和 App Secret
  3. 点击「获取群列表」按钮，选择要监听的群（自动填入 Chat ID）
  4. 勾选「启用消息轮询」
  5. 点击「保存飞书配置」
  6. 点击「发送测试消息」验证配置是否正确

🔌 方式三：长连接 WebSocket（推荐，替代轮询）

通过飞书 WebSocket 长连接实时接收消息，比轮询延迟更低、资源消耗更小。 配置步骤与方式二完全相同（共用 App ID/App Secret），区别在于：

  • 在「权限管理」中额外开启 im:resource 权限（接收消息资源）
  • 启用「消息轮询」后，系统自动使用长连接代替轮询
  • 无需额外配置，应用启动后自动连接飞书 WebSocket 网关

🗂️ 数据文件

所有数据以 JSON 文件形式存储在笔记库根目录下，无数据库依赖。 以下路径均为相对于笔记库根目录：

文件读写说明
{config-dir}/config.json读写应用配置（飞书、笔记库路径、模块定义等）
{config-dir}/assistant_log.json读写操作日志（最多保留 100 条）
{config-dir}/memory.db读写SQLite 数据库（对话记录）
AI/综合日报/读写AI 生成的综合日报文件（按日期命名）
AI/任务/读写任务数据
AI/提醒/reminders.json读写定时提醒数据
AI/prompts/读写提示词文件（综合日报提示词等）
AI/记忆/读写AI 记忆文件
AI/collector/读写采集器配置和提示词
代办.md读写每日循环和临时提醒
