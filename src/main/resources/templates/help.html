📖 AI 助理 v3.0 帮助

🚀 快速开始

AI 助理 v3.0 基于 Spring AI 2.0，直接调用 LLM API（如 DeepSeek），不再需要外部编排服务。

✅ 前置依赖

依赖说明是否必须
Java 17+运行时✅ 必须
Ollama语义检索加速（可选）🔘 可选
LLM API KeyDeepSeek 或其他兼容 API✅ 必须

⚡ 启动步骤

# 1. 配置 LLM API Key（在 Web 界面 /config 页面设置，或设环境变量）
#    DEEPSEEK_API_KEY=sk-xxxxx

# 2. 启动应用
java -jar target/ai-assistant-3.0.0.jar

# 3. 访问 http://localhost:6790

🅰️ 启动后先去 /config 页面配置笔记库根目录和 LLM API Key。

💬 对话模块

  • 知识库对话（默认）：结合笔记库历史和语义检索回答问题
  • 创作式对话：开启新会话，AI 无历史上下文
  • 笔记操作：直接让 AI 读/写笔记库文件，如"帮我记录一条 BUG"

📊 模块（业务数据）

在笔记库中设置模块，每个模块可配置数据文件（JSON/CSV），AI 自动分析数据给出洞察。

📸 图片识别

利用 LLM 的多模态（Vision）能力分析笔记库中的图片。支持 PNG/JPG/GIF/WebP/BMP。

☀️ 日报系统

  • 工作日 09:00：自动生成综合日报（基于工作笔记 + 语义记忆）
  • 工作日 18:00：下班提醒推送到飞书
  • 日报存储在 AI/综合日报/ 目录

📡 采集器（数据收集）

定时从指定 URL 采集数据，AI 分析提取为结构化 JSON，存入数据中心。

🔧 配置

所有配置在 /config 页面管理，无需手动编辑文件。

配置项说明
笔记库根目录所有笔记、业务数据、日报等文件的存放目录
API KeyLLM API 密钥（DeepSeek 等）
飞书集成Webhook URL 和应用凭证，用于消息推送
模块配置业务数据模块的目录和文件定义

🆘 常见问题

问题原因解决
AI 回复"API Key 未配置"未在 /config 填写 API Key在配置页填写 LLM API Key
AI 回复为空LLM 返回内容为空或超时检查 API Key 和网络，超时可设为 600s
语义检索不可用Ollama 未启动启动 ollama run nomic-embed-text

🏗️ 技术架构

  • Spring Boot 4.1 + Spring AI 2.0 + Java 17
  • AI 引擎：Spring AI ChatClient 直连 LLM（DeepSeek API）
  • 工具编排：@Tool 注解 + ToolCallingAdvisor 自动管理多轮工具调用
  • 语义检索：Spring AI OllamaEmbeddingModel (nomic-embed-text)
  • 数据存储：SQLite (MyBatis-Plus) + JSON 文件
  • 前端：Thymeleaf 模板
