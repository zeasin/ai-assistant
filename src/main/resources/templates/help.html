快速开始

AI 助理 v0.4.0 基于 Spring AI 2.0 直连 LLM API，无需外部编排服务。启动后访问 /config 配置笔记库根目录和 API Key 即可使用全部功能。

前置依赖

依赖说明必须
Java 17+运行时环境✅
LLM API KeyDeepSeek 等兼容 API✅
语义检索向量嵌入（支持 Ollama 本地 / 硅基流动等 API）🔘 可选

启动步骤

# 1. 配置 API Key（/config 页面或环境变量 DEEPSEEK_API_KEY）
# 2. 启动应用
java -jar target/ai-assistant-0.4.0.jar
# 3. 访问 http://localhost:6790

提示：先配 API Key 和笔记库路径，否则 AI 功能和文件操作不可用。

AI 对话

支持多轮对话，包含历史上下文（同会话）和跨会话语义检索（需 Ollama）。AI 可自主调用工具操作笔记库。

工作模式

  • 知识库对话（默认）— 注入历史上下文 + 语义检索，适合基于笔记库的问答
  • 创作式对话 — 无历史上下文，适合独立创作或开放讨论

SSE 实时流式响应

发送消息后，后端通过 Server-Sent Events 推送到前端，事件类型：

事件说明
session建立会话，返回 sessionId，前端更新 URL
status进度指示（处理中/读取文件/写入等）
textAI 回复文本
done流式完成
error错误信息

笔记工具（@Tool 自动编排）

AI 根据对话内容自主决定是否调用以下工具，无需手动指定：

  • listDir(path) — 列出笔记库目录内容
  • readFile(path) — 读取文件（上限 8000 字符）
  • writeFile(path, content) — 写入文件
  • searchFiles(keyword) — 按文件名搜索（最多 20 条）

语义检索 / 长期记忆

配置页「语义向量模型」卡片配置嵌入模型。支持两种模式：

  • Ollama 本地 — 不填 API Key，使用本地 Ollama（推荐 nomic-embed-text）
  • API 模式 — 填入 API Key，支持阿里云百炼、硅基流动等 OpenAI 兼容服务

AI 回复后自动生成向量并跨会话检索相似轮次（余弦相似度阈值 0.5），为当前对话提供相关历史背景。最多从 5 轮最相关的历史中提取上下文。向量模型不可用时回退最近 3 轮对话。

对话管理

  • 左侧历史会话列表，支持切换、删除
  • 消息自动保存至 SQLite，刷新页面不丢失
  • 导出对话为 Markdown
📁

业务数据模块

工作模块是业务数据的组织单元。每个模块包含数据文件（JSON）、文档树、AI 分析提示词。模块在 /config 页面配置。

模块配置（config.json）

{
  "modules": [
    {"id": "self-media", "name": "自媒体",
     "dir": "自媒体", "icon": "📝",
     "dataFiles": ["data.json"]},
    {"id": "customer", "name": "客户管理",
     "dir": "企业/客户管理", "icon": "👥",
     "dataFiles": ["data.json"]}
  ]
}

功能

  • 数据表格 — 查看、新增、编辑、删除记录（支持 JSON 嵌套结构）
  • 文档浏览 — 模块目录下的 Markdown 文件树，在线预览
  • AI 分析 — SSE 流式分析模块数据并生成洞察报告
  • 自定义提示词 — 每个模块可配置专属 AI 分析提示词
  • 列设置 — 自定义表格列显示名称、顺序和可见性
  • 字段标签 — 在配置页为英文 key 设置中文标签
🗄️

数据中心

结构化数据管理平台，支持多数据集的定义、导入、导出和搜索。

数据集管理

  • 创建带 Schema 的数据集（字段名、类型、显示名、必填、默认值）
  • 记录浏览、搜索（关键词匹配）、删除
  • 自动 MD5 去重

导入方式

  • Excel 导入 — 自动检测列头、模糊匹配字段映射，支持 AI 智能归一化
  • JSON 导入 — 直接粘贴或上传 JSON 数据
  • URL 导入 — AI 自动访问 URL 并提取结构化数据

导出

将数据集记录导出到工作模块，AI 自动转换为模块格式。

📡

数据采集器

定时采集外部数据并提取为结构化 JSON，支持动态 Cron 调度。

任务类型

  • AI 提示模板 — 发送提示词给 LLM，从回复中提取 JSON
  • URL 抓取 — AI 访问 URL 并提取数据

功能

  • 每个任务独立配置 Cron 表达式
  • 执行结果自动去重（MD5）
  • 可关联数据集自动导入
  • 执行日志查看
  • 默认内置"CSDN 数据采集"任务
📸

图片识别

利用 LLM 多模态（Vision）能力分析笔记库中的图片。

  • 从笔记库目录浏览并选择图片
  • 直接上传图片进行分析
  • 支持格式：PNG / JPG / GIF / WebP / BMP
  • 分析结果可保存到笔记库
📋

任务看板

任务数据存储在 AI/任务/data.json。

  • 任务字段：标题、描述、状态（待办/完成）、优先级（高/中/低）、截止日期
  • 首页侧边栏按优先级显示未完成任务
  • 支持新增、编辑、删除
⏰

定时提醒

通过飞书推送定时提醒，支持多种重复类型。

提醒类型

  • 一次性 — 指定日期时间触发
  • 每日 — 每天固定时间
  • 每周 — 每周指定日期的固定时间
  • 每月 — 每月指定日期的固定时间
  • 每年 — 每年指定日期的固定时间

内置提醒

  • 工作日 18:00 — 下班日报提醒（推送飞书）

系统每分钟检查一次到期提醒，触发后通过飞书 Webhook 推送。

综合日报

AI 自动生成综合日报，整合工作笔记、当日日报、AI 记忆文件等内容。

自动生成（工作日 09:00）

  • 读取 AGENTS.md（前 3000 字符）了解项目规则
  • 读取 AI 记忆文件了解近期上下文
  • 读取当日工作日报
  • 调用 LLM 生成综合日报并保存至 AI/综合日报/

自动推送（工作日 18:00）

通过飞书 Webhook 推送下班日报提醒。

手动操作

  • 首页一键触发日报生成
  • 自定义分析提示词

  • Markdown 预览 — 渲染为 HTML 查看

AI 指南

管理 AI 对话的规则文件（AGENTS.md）和长期记忆文件。

AGENTS.md

AI 在对话中每次都会读取的规则文件，定义笔记库目录结构、数据格式、操作规范。可在页面直接编辑保存。

AI 记忆文件

存储在 AI/记忆/ 目录。在日报生成和对话语义检索中被引用。支持查看、编辑、新建、删除。

✈️

飞书集成

支持飞书 Webhook 消息推送和 Bot API 消息接收与自动回复。

Webhook 推送

  • 日报推送、下班提醒、文章提醒
  • 富文本格式（标题 + 段落 + 引用）

Bot API（自建应用）

  • 配置 App ID / App Secret / Chat ID
  • WebSocket 长连接接收群聊/单聊消息（需应用被 @ 或被加好友）
  • AI 自动回复（同对话系统，飞书消息作为独立会话管理）
  • 启用手动切换，默认关闭
⚙️

系统配置

所有配置通过 /config 页面管理，无需手动编辑文件。配置存储在 config.json。

配置项说明
LLM API KeyDeepSeek 等兼容 API 密钥（必须）
API 地址兼容 OpenAI 格式的 API 地址
模型名如 deepseek-chat
超时时间建议 ≥300 秒
笔记库根目录所有业务数据的存放根目录（必须）
飞书 Webhook URL消息推送 URL
飞书应用凭证App ID / Secret，用于 Bot API
字段标签英文 key → 中文标签映射

配置优先级

Web 页面配置 > 环境变量（DEEPSEEK_API_KEY / LLM_API_KEY / LLM_BASE_URL / LLM_MODEL）> 默认值

技术架构

组件技术
框架Spring Boot 4.1.0 + Spring AI 2.0.0 + Java 17
AI 引擎ChatClient 直连 LLM（OpenAI 兼容协议）
工具编排@Tool 注解 + ToolCallingAdvisor（自动路由）
语义检索Ollama（nomic-embed-text 等）或 API（BAAI/bge-m3 等）

前端Thymeleaf 模板
飞书集成HttpURLConnection（Webhook）+ Lark SDK（WebSocket）

常见问题

问题原因解决
AI 回复"API Key 未配置"未在 /config 填写 API Key配置页填写 LLM API Key
AI 回复为空LLM 超时或返回空检查网络，超时设为 600s
"未配置笔记库根目录"未配置笔记库路径在 /config 页面设置
语义检索不可用向量模型未配置或服务未启动/config 页面配置语义向量模型
