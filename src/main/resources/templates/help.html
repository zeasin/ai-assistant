📖 AI 助理 (ai-assistant)

一站式个人工作台，集成 AI 综合日报、对话问答、笔记管理、公众号运营分析、客户管理等功能。 所有 AI 能力基于本地笔记库和 opencode serve，不依赖任何第三方云服务。 开源版免费，商业版即将推出。

🚀 快速启动

环境要求

依赖版本用途
Java≥17Web 服务运行环境
opencodelatestAI 引擎（必须启动两个实例）

启动步骤

# 1. 先启动两个 opencode serve 服务（必须）
opencode serve --port 14096  # AI 聊天/笔记库服务
opencode serve --port 14099  # 代码分析服务

# 2. 编译项目
mvn package -q

# 3. 启动应用
java -jar target/ai-assistant-2.0.0.jar

# 访问地址
# - Web UI: http://localhost:6790
# - 健康检查: http://localhost:6790/health

系统依赖两个 opencode serve 实例提供所有 AI 能力，缺一不可：

⚙️ 首次使用配置

首次启动后，请先前往 配置页 完成以下设置：

  1. 笔记库根目录 — 设置笔记库的绝对路径（如 D:\projects\richie_learning_notes），所有数据文件的相对路径都基于此目录
  2. 工作目录 — 默认 工作，日报/周报保存在 {根目录}/工作/日报/ 和 {根目录}/工作/周报/
  3. 客户数据目录 — 默认 企业/客户管理，客户、线索、跟进记录数据
  4. 运营数据目录 — 默认 自媒体，公众号运营数据
  5. 飞书 Webhook — 可选，配置后可接收推送通知
  6. 字段标签 — 可选，自定义数据表格中英文 key 显示为中文标签

未配置前，任何需要访问笔记库的操作会提示你去配置。

📊 综合日报

首页展示 AI 综合日报，数据源为笔记库中的记忆文件，涵盖：项目进展、客户动态、今日 TODO、昨日回顾、学习方向、生活提醒、个人目标、AI 建议。 日报文件保存在 {笔记库根目录}/工作/综合日报/{日期}.md，可由定时任务自动生成，也支持 AI 对话手动触发。

📋 工作看板

工作看板用于管理日报和周报，支持添加、编辑、AI 分析工作趋势。

  • 日报管理 — 添加/编辑日报，支持逐条添加工作任务和明日计划
  • 周报管理 — 添加/编辑周报，Markdown 格式自由编辑
  • AI 分析 — 基于近期日报和周报，AI 自动生成工作趋势分析和改进建议

数据存储：工作/日报/{日期}.md 和 工作/周报/{年 - 周}.md

💬 AI 对话

与 AI 助理对话，它会读取整个笔记库来回答问题。支持流式响应、实时状态显示、历史会话管理。

路由规则

输入路由AI 引擎
普通文字（默认）📚 知识库AI14096 端口 opencode（笔记库）
/code 你的问题🔧 编程AI14099 端口 opencode（代码分析）

注意：路由规则基于消息前缀，而非关键词匹配。不以 /code 开头的消息一律走知识库AI。 前缀在发送给 AI 前会自动剥离，AI 不会看到 /code 前缀。

核心特性

  • 路由规则 — 默认知识库，/code 前缀触发编程AI，互不干扰
  • 无限等待 — 不设超时，AI 思考多久等多久，无需重试
  • 流式响应 — 回答逐字显示，无需等待全部生成完毕
  • 实时状态 — AI 读取笔记时显示 📖 文件名，知道它在做什么
  • 历史会话 — 左侧边栏保存所有对话记录，可切换、可删除
  • 历史会话 — 左侧边栏保存所有对话记录，可切换、可删除

📝 快速数据录入

核心原则：规则由你在 AI 的 AGENTS.md 中制定，AI 自己保存数据。 Java 代码只做消息转发，不参与任何业务逻辑。

你只需在笔记库根目录的 AGENTS.md 中定义录入规则（字段映射、计算方式、保存格式等）， 所有入口的消息都会原样发给 AI，AI 按你的规则解析并直接写入数据文件。

入口方式说明
页面表单运营/客户页面点按钮弹窗填写字段拼成文本发给 AI，AI 按规则保存
工作看板日报/周报页面直接填写直接保存到文件，无需 AI 处理
飞书消息以"记录/新增/添加/创建/录入"开头发消息原样转发 AI，AI 按规则保存并回复
AI 对话聊天框以"记录/新增/添加/创建/录入"开头输入原样转发 AI，AI 按规则保存并回复

📩 飞书集成

支持三种飞书集成方式，可根据需要组合使用。所有配置均在 配置页 中设置。

🔗 方式一：Webhook 推送（接收通知）

配置 Webhook URL 后，AI 综合日报、发文提醒、下班提醒等自动推送到飞书群。
仅需群机器人权限，配置最简单。

  1. 打开飞书，进入目标群聊的「设置」→「群机器人」→「添加机器人」
    权限：无需申请，群管理员可直接添加
  2. 搜索并添加「自定义机器人（Webhook）」
  3. 复制机器人 Webhook 地址，格式：https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx
  4. 进入本系统 配置页，在「飞书 Webhook 配置」中粘贴地址并保存
  5. 点击「手动触发」区域的按钮（如🌅生成综合日报）测试推送是否正常

📡 方式二：消息轮询（API 方式接收消息）

通过飞书 API 定时轮询指定群的未读消息，AI 自动回复。
需创建飞书自建应用，配置复杂但功能完整。

第一步：创建飞书自建应用

  1. 打开 飞书开发者后台，点击「创建企业自建应用」
    权限：需要飞书管理员账号或拥有创建应用权限
  2. 填写应用名称（如：AI 助理）、描述，创建完成后进入应用详情

第二步：获取 App ID 和 App Secret

  1. 在应用详情页左侧菜单点击「凭证与基础信息」
    权限：应用创建者或管理员可查看
  2. 复制 App ID 和 App Secret（App Secret 首次需点击「显示」或重置）

第三步：配置权限

  1. 在左侧菜单点击「权限管理」
  2. 开启以下权限（必选）：
    • im:message — 获取单聊、群组消息（read_message 和 send_message 子权限）
    • im:chat — 获取群列表信息（read_chat 子权限）
  3. 点击「批量开通」并确认

第四步：发布应用

  1. 在左侧菜单点击「版本管理与发布」
  2. 创建版本，填写版本号（如 1.0.0）、更新说明，点击「保存」
  3. 点击「申请发布」，选择审核方式（推荐「普通发布」，需管理员审批）
  4. 联系飞书管理员审批通过

第五步：将应用添加到群聊

  1. 打开要监听的飞书群，进入「设置」→「群机器人」→「添加机器人」
    权限：群管理员可操作
  2. 搜索并添加刚才创建的自建应用

第六步：在系统中配置

  1. 进入本系统 配置页，在「飞书消息接收（API 轮询）」区域
  2. 填入 App ID 和 App Secret
  3. 点击「获取群列表」按钮，选择要监听的群（自动填入 Chat ID）
  4. 勾选「启用消息轮询（每 15 秒检查一次）」
  5. 点击「保存飞书配置」
  6. 点击「发送测试消息」验证配置是否正确

🔌 方式三：长连接 WebSocket（推荐，替代轮询）

通过飞书 WebSocket 长连接实时接收消息，比轮询延迟更低、资源消耗更小。 配置步骤与方式二完全相同（共用 App ID/App Secret），区别在于：

  • 在「权限管理」中额外开启 im:resource 权限（接收消息资源）
  • 启用「消息轮询」后，系统自动使用长连接代替轮询
  • 无需额外配置，应用启动后自动连接飞书 WebSocket 网关

注意：长连接模式下，应用需保持运行。如果网络断开，系统会自动重连。

🎯 消息路由规则（长连接模式）

与 Web 聊天页面规则一致，所有消息默认走知识库，仅以 /code 开头的消息触发编程 AI。 前缀在发送给 AI 前自动剥离。

飞书消息路由说明
今天对接了1个新客户...📚 知识库AI默认走笔记库，读取笔记库数据回答
/code 帮我写个用户管理...🔧 编程AI走代码分析引擎，访问项目代码

聊天上下文隔离

知识库对话和编程对话的聊天记录按模式分开保存。知识库历史不会混入编程上下文，反之亦然。 当 opencode session 过期重建时，自动注入对应模式的最近历史记录，保持对话连贯性。

👥 客户管理

客户管理看板展示概览、销售漏斗、合同金额、线索漏斗、近期活动、待跟进等数据。 数据存储在 {笔记库根目录}/企业/客户管理/ 目录，支持通过 AI 对话录入和更新。

⏰ 定时提醒

定时提醒功能支持创建和管理各种循环提醒任务，到时间自动推送消息到飞书。

  • 每天 — 每天固定时间提醒，如每日站会、下班日报
  • 每周 — 每周特定星期几提醒，如周例会、周报
  • 每月 — 每月特定几号提醒，如月度总结、账单日
  • 每年 — 每年特定日期提醒，如生日、纪念日

支持启用/禁用、编辑、删除、立即触发等操作。提醒触发后会推送到配置的飞书群。

  • 删除 — 删除 .md 文件
