# 重启应用验证调度任务注册

## 已完成的代码修改
- `SchedulingConfig.java` — 将 `ThreadPoolTaskScheduler` 暴露为 `@Bean`
- `CollectorScheduler.java` — `CronTrigger` 添加 `ZoneId.of("Asia/Shanghai")`，增加启动详细日志

## 执行步骤

1. 启动应用（`mvn spring-boot:run`）
2. 观察控制台日志，确认以下输出：
   - `CollectorScheduler initializing...`
   - `Found N total collector tasks, checking enabled ones...`
   - `Task: ... enabled=true, cron=...`
   - `Scheduled collector task ... with cron: ...`
   - `Rescheduled N collector tasks`
3. 调用 API 验证调度状态：`GET /api/collector/tasks`
4. 检查所有定时任务日志：`GET /api/collector/logs`

## 验证标准
- 控制台日志显示任务已成功注册到调度器
- API 返回任务列表，enabled=true 的任务显示在已调度列表中
