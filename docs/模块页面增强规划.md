# 模块页面（kb_module_detail）增强规划

> 版本：v1.1
> 状态：规划中
> 日期：2026-06-19

---

## 一、概述

模块页面本质是笔记库子文件夹的分析界面。当前已有 AI 分析、原始数据、文档查看、提示词四个功能，需要补充笔记管理、搜索、数据可视化、文件上传等核心能力。

---

## 二、新增功能清单

### 2.1 新建笔记

| 项目 | 说明 |
|------|------|
| 入口 | 模块详情页新增"📝 新建笔记"按钮 |
| 功能 | 在模块目录下创建新的 Markdown 文件 |
| 表单 | 文件名（必填）、内容（Markdown 编辑器） |
| API | `POST /module/{id}/docs/create` |
| 参数 | `name`（文件名）、`content`（内容）、`path`（子目录，可选） |
| 保存位置 | `{module.dir}/{path}/{name}.md` |

### 2.2 笔记检索（向量搜索）

| 项目 | 说明 |
|------|------|
| 入口 | 模块详情页新增"🔍 搜索"tab 或搜索框 |
| 功能 | 在模块所有文件中搜索关键词，支持语义搜索 |
| 实现方式 | 两阶段：先实现文件名 + 内容关键词搜索；后续接入向量搜索 |
| API | `GET /module/{id}/search?q=关键词` |
| 返回 | 匹配的文件列表 + 匹配片段高亮 |

**第一阶段（简单搜索）：**
- 遍历模块目录下所有 .md / .txt / .json 文件
- 按文件名和内容匹配关键词
- 返回文件路径 + 匹配行

**第二阶段（向量搜索）：**
- 复用 `turn_embeddings` 表结构，或新建 `note_embeddings` 表
- 对模块内笔记文件进行分块向量化
- 查询时将搜索词向量化，余弦相似度匹配

### 2.3 数据可视化

| 项目 | 说明 |
|------|------|
| 入口 | 模块详情页新增"📊 可视化"tab |
| 功能 | 对模块内 JSON 数据生成图表 |
| 图表类型 | 柱状图、饼图、折线图（基于数据自动判断） |
| 技术方案 | 使用 Chart.js（CDN 引入，无需 npm） |
| API | `GET /module/{id}/chart-data?file=xxx&group=xxx` |
| 返回 | 适配 Chart.js 的 labels + datasets 格式 |

**自动判断逻辑：**
- 数值字段 → 柱状图 / 折线图
- 分类字段 → 饼图
- 时间 + 数值 → 折线图

### 2.4 文件上传

| 项目 | 说明 |
|------|------|
| 入口 | 模块详情页新增"📤 上传文件"按钮 |
| 功能 | 上传文件到模块目录，可选同步生成 AI 分析 |
| API | `POST /module/{id}/upload` (multipart/form-data) |
| 参数 | `file`（文件）、`path`（子目录，可选）、`analyze`（是否生成分析，默认 false） |
| 支持格式 | .md, .txt, .json, .csv, .xlsx, .png, .jpg |
| 上传后 | 自动刷新数据列表 |

**AI 分析同步：**
- 上传 JSON 数据文件时，可选自动触发 AI 分析
- 上传后调用 `/module/{id}/analysis?force=true` 生成分析报告

## 七、技术要点

### 新建笔记 API

```java
@PostMapping("/{id}/docs/create")
@ResponseBody
public Map<String, Object> createDoc(@PathVariable String id,
                                      @RequestBody Map<String, String> body) {
    String name = body.get("name");
    String content = body.getOrDefault("content", "");
    String path = body.getOrDefault("path", "");
    
    ModuleDefinition mod = moduleService.getModule(id);
    Path dir = moduleService.getModuleDir(mod);
    Path target = dir.resolve(path).resolve(name + ".md").normalize();
    
    // 安全校验
    if (!target.startsWith(dir)) return Map.of("ok", false, "error", "路径越界");
    
    Files.createDirectories(target.getParent());
    FileUtil.writeText(target, content);
    return Map.of("ok", true, "path", path + "/" + name + ".md");
}
```

### 搜索 API

```java
@GetMapping("/{id}/search")
@ResponseBody
public Map<String, Object> search(@PathVariable String id, @RequestParam String q) {
    ModuleDefinition mod = moduleService.getModule(id);
    Path dir = moduleService.getModuleDir(mod);
    
    List<Map<String, Object>> results = new ArrayList<>();
    Files.walk(dir, 10)
        .filter(p -> isSearchable(p))
        .forEach(p -> {
            String content = FileUtil.readText(p);
            if (content.contains(q)) {
                results.add(Map.of(
                    "path", dir.relativize(p).toString(),
                    "snippet", extractSnippet(content, q)
                ));
            }
        });
    
    return Map.of("ok", true, "results", results);
}
```
