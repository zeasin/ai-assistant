## 一、日报增强

### 1.1 现状

ReportService 已经走对了方向（AgentAnalysisService + AI 自主 @Tool 探索笔记库），
但引导不足：AI 不知道今天哪些文件变了，不知道知识库结构，没有趋势对比。

```
现在的流程:
  Prompt 硬编码模板 → AI 盲搜关键词 → 找到什么算什么 → 拼日报

改后的流程:
  1. scanTodayChanges()   扫笔记库 mtime → 今天哪些文件变了（客观）
  2. 读 AGENTS.md         → 知识库规则和结构
  3. 读昨日日报 + 数据集对比 → 趋势变化数字
  4. 构造增强 Prompt      （地图 + 规则 + 趋势）
  5. AgentAnalysisService  AI 自主 @Tool 探索笔记库
  6. 保存 Markdown + 推飞书
```

### 1.2 scanTodayChanges()

```java
private List<FileChange> scanTodayChanges(Path kbDir, String notesDir) {
    // 1. 递归遍历笔记库所有文件
    // 2. 按 mtime 筛选今天修改/创建的
    // 3. 跳过隐藏目录（.git, .idea, .obsidian 等）
    // 4. 跳过 AI/综合日报/ 历史日报
    // 5. 标注：相对路径 / 所属模块（路径推断）/ 类型（json|md|code|image）
    //          变更类型（new|modified）/ 文件大小 / 和昨天对比的 sizeDelta
    // 6. 按模块分组
}

public record FileChange(
    String path,          // "客户/张三/data.json"
    String module,        // "客户"
    String type,          // "json"
    String changeType,    // "modified"
    long size,
    long sizeDelta,
    String summary
) {}
```

## 二、工作场景串联（数据模块 + 对话 + 日报）

### 2.1 核心思想

数据模块和日报不是孤立功能。它们和"对话"一起，构成一天工作的**完整闭环**：

```
  早上/白天：干活 → 原始数据产生 → 清洗 → 进入笔记库（干净数据沉淀）
  全程随时：对话（万能入口，直接读写笔记库）
  晚上：日报（从笔记库事实自动汇总分析，不是瞎编）
```

**对话是万能入口**——对话不读 SQLite，直接读笔记库（因为笔记库才是干净、最终的数据）。

### 2.2 一天的工作流（真实走一遍）

```
═══════════════════════════════════════════════════════════
场景一：早上处理 Excel 客户名单（数据模块 → 笔记库）
═══════════════════════════════════════════════════════════

  我：数据工作台 → Tab1 原始仓库 → 上传 Excel → 得到 47 条原始记录
  我：Tab2 清洗工厂 → 选这个数据集 → 勾选 normalize+validate+dedup
      → 预览 47 条去重后 38 条
  我：Tab3 进入笔记库 → 保存文件对话框
      选知识库"工作" → 浏览到客户/data/ → 文件名 data.json
      格式 (.json) → 预览: 已有 47 条 + 新增 38 条去重合并 = 85 条
  我：确认写入 → 笔记库/客户/data/data.json 更新了
  ✅ 之后对话可以直接 readFile 读到这 85 个客户


═══════════════════════════════════════════════════════════
场景二：白天和 AI 对话（直接读写笔记库，万能入口）
═══════════════════════════════════════════════════════════

  我："帮我查一下启航电商ERP的客户有哪些？"
  AI：（自动）readFile("客户/data/data.json") → 启航ERP有 3 个客户...
  我："再加一个客户，叫老王，电话 138..."
  AI：（自动）writeFile → 追加到客户/data/data.json 末尾
  ✅ 对话直接操作笔记库文件，不经过数据工作台
  ✅ 数据工作台稍后点刷新就能看到这条新客户（递归扫描找到）


═══════════════════════════════════════════════════════════
场景三：Collector 爬数据（自动化 → 数据模块 → 笔记库）
═══════════════════════════════════════════════════════════

  我：设置每天早上 6:00 自动爬 CSDN
  Collector：（自动）抓到 128 篇 → 自动进 原始仓库 待清洗
  飞书推："采集完成：CSDN 抓到 128 条原始数据，等待清洗"
  我（晚上）：数据工作台 → 清洗工厂 → 清洗掉广告/重复 → 112 条干净
  我：进入笔记库 → 工作/AI/数据/CSDN文章/records.jsonl → .jsonl 格式
  ✅ 之后日报会自动看到 CSDN文章 今天 +112


═══════════════════════════════════════════════════════════
场景四：晚上一键生成日报（日报增强版的能力）
═══════════════════════════════════════════════════════════

  我：点"生成今日日报"
  ReportService 改造后流程：
    1. scanTodayChanges(工作笔记库)
       → 今天哪些文件变了？客户/data/data.json (+38条),
         AI/数据/CSDN文章/records.jsonl (+112),
         AI/任务/data.json (+3 完成), ...
    2. 读 AGENTS.md → 知识库规则
    3. 对比昨日 → 今日 +4 文件模块 +150 记录
    4. 构造增强 Prompt（注入上面 3 条事实）
    5. AI AgentAnalysisService 自主 @Tool：
         readFile("客户/data/data.json") → 分析客户变化
         readFile("AI/数据/CSDN文章/records.jsonl") → 分析新文章趋势
         readFile("AI/任务/data.json") → 哪些任务完成了
    6. 写 Markdown + 推飞书
  ✅ 日报所有结论都来自笔记库真实文件，不是瞎编
```

### 2.3 三者的关系

```
                  ┌───────────────────────┐
                  │     数据工作台          │
                  │  (原始仓库/清洗工厂)    │
                  └───────────┬─────────────┘
                              │ DataIngressService
                              ▼
┌──────────────┐   readFile/writeFile   ┌───────────────┐   readFile   ┌──────────────┐
│   对话模块    │ ─────────────────────▶ │    笔记库     │ ◀────────── │  ReportService│
│  (万能入口)   │ ◀────────────────────── │  (纯净中心)    │ ─────────── │  (日报增强)   │
└──────────────┘                          └───────────────┘             └──────┬───────┘
                                                                                 │
                                                                                 ▼
                                                                          综合日报 Markdown
                                                                          飞书推送
```

**笔记库是唯一中心**——所有数据最终汇聚到这里，所有读取都从这里来。

```
写入笔记库的方式（两条路）：
  A. 数据工作台 UI（Excel/URL/Collector/JSON）
     原始仓库 → 清洗工厂 → 进入笔记库(保存文件对话框) → 笔记库
  B. 对话直接录（AI @Tool writeFile）
     用户对话 → AI 用 writeFile → 直接写笔记库任意位置
     （不变，不经过数据工作台）

读取笔记库的方式（两条路）：
  A. 对话 AI @Tool readFile → 直接回答
  B. ReportService 扫 mtime + AI AgentAnalysis → 日报
```

### 2.4 这次改了什么（工作场景串联维度）

```
之前的痛点           → 改后
────────────────────────────────────────────────────────────────
数据进不了笔记库     → 数据工作台完整链路打通
  （原始仓库只存不进）     （上传→清洗→进入笔记库 一条线）

日报瞎编             → 日报基于事实
  （硬编码 Prompt）       （scanTodayChanges 扫笔记库 mtime + AI 读真实文件）

对话不知道数据在哪    → 对话直接读笔记库
  （查 SQLite 或搜关键词）  （笔记库才是干净数据源）

数据散在各处看不到    → 数据工作台总览
  （客户/项目/提醒分模块）   （统一视图 + 知识库切换 + 图表）

同名文件处理没流程    → 保存文件对话框
  （不知道会不会覆盖）       （合并去重/覆盖/重命名 + 预览）
```
