## 1. 启动系统

### 环境要求

| 组件 | 要求 | 说明 |
|------|------|------|
| JDK | 17+ | 必装 |
| Maven | 3.8+ | 仅编译需要，运行不需要 |
| LLM API Key | 必填 | DeepSeek / 商汤 / 智谱等 |
| Ollama | 可选 | 本地语义检索用 |

### 编译启动

```bash
# 1. 编译
mvn package -q

# 2. 启动
java -jar target/ai-assistant-1.0.0.jar

# 3. 浏览器打开
open http://localhost:6790
```

### 启动后第一件事

浏览器打开 `http://localhost:6790`，系统会自动跳转到配置页面。如果你是第一次启动，需要按以下顺序完成配置：

> **配置顺序**：LLM 模型 → 笔记库 → Embedding（可选）

## 2. 配置 LLM 模型

AI 对话的核心依赖，必须先配置。

### 2.1 配置默认模型

1. 打开左侧菜单 **⚙️ 配置**
2. 找到 **AI 模型配置** 区域
3. 填写以下三项：

| 字段 | 示例值 | 说明 |
|------|--------|------|
| API Key | `sk-xxxxxxxxxxxxxxxx` | 从模型提供商获取 |
| Base URL | `https://api.deepseek.com` | API 地址（**不要以 /v1 结尾**） |
| 模型名 | `deepseek-chat` | 具体模型名称 |

4. 点击 **保存**

> Base URL 常见值：
> - DeepSeek：`https://api.deepseek.com`
> - 智谱 GLM：`https://open.bigmodel.cn/api/paas/v4`
> - 商汤日日新：`https://token.sensenova.cn/v1`
> - Ollama 本地：`http://127.0.0.1:11434/v1`
> - 硅基流动：`https://api.siliconflow.cn/v1`

### 2.2 添加多个模型（可选）

支持配置多个模型，在对话时切换使用：

1. 在配置页点击 **添加模型**
2. 填写模型信息，选择模型类型：

| 类型 | 说明 | 用途 |
|------|------|------|
| `text` | 文本模型 | 对话、分析、工具调用 |
| `multimodal` | 多模态模型 | 图片识别（需支持 vision） |
| `embedding` | 向量模型 | 语义检索（见第 5 节） |

3. 勾选 **设为默认** 可将该模型设为常用模型

### 2.3 测试连接

保存后，回到 **💬 对话** 页面发送一条消息，如果能正常回复说明配置成功。

## 3. 配置笔记库

笔记库是 AI 的知识库，所有笔记文件（Markdown / JSON / TXT）都从这个目录读取。

### 3.1 创建第一个笔记库

1. 打开左侧菜单 **⚙️ 配置**
2. 找到 **笔记库管理** 区域
3. 点击 **新建笔记库**
4. 填写：

| 字段 | 示例 | 说明 |
|------|------|------|
| 名称 | 工作笔记 | 笔记库的显示名称 |
| 笔记库路径 | `/Users/me/notes` | 笔记文件所在的本地目录（绝对路径） |

5. 点击 **保存**

> 笔记库路径**必须是一个已存在的本地文件夹**，里面可以放 `.md`、`.json`、`.txt` 文件。

### 3.2 多笔记库管理

支持创建多个笔记库（如：工作/学习/项目），数据完全隔离：

- 每个笔记库有独立的笔记文件
- 聊天历史按笔记库隔离
- 任务/提醒按笔记库隔离
- AI 可以跨笔记库切换操作

操作方式：
- 左侧侧边栏会列出所有笔记库
- 点击笔记库名称即可切换
- 在对话中也可以让 AI 切换：_"切换到工作笔记库"_

### 3.3 笔记库目录设置

每个笔记库可以：

- **排序**：自定义目录显示顺序
- **隐藏目录**：不显示特定文件夹
- **排除索引**：指定文件夹不参与语义搜索

在 **AI 中心** → **目录管理** 中操作。

## 4. 配置语义检索（向量）

> **这一步决定 AI 能否"理解"笔记内容进行搜索。** 如果不配置，AI 只能按文件名搜索，不能按内容理解来搜索。

### 4.1 方式一：Ollama 本地（推荐，免费）

适用于 Mac / Linux 用户：

```bash
# 1. 安装 Ollama
brew install ollama

# 2. 启动
ollama serve

# 3. 拉取向量模型
ollama pull bge-m3

# 4. 刷新 Embedding 状态
# 在配置页点击"测试连接"或重启系统
```

### 4.2 方式二：在线 API

适用于不想本地跑 Ollama 的用户：

1. 在配置页找到 **语义检索（向量）** 区域
2. 选择 **API 模式**
3. 填写：

| 字段 | 示例 | 说明 |
|------|------|------|
| 提供商 | `siliconflow` | 仅用于显示标识 |
| Base URL | `https://api.siliconflow.cn/v1` | API 地址 |
| API Key | `sk-xxxx` | 从提供商获取 |
| 模型名 | `BAAI/bge-m3` | 向量模型名称 |

### 4.3 构建索引

配置好 Embedding 后，需要为笔记库构建索引：

1. 打开 **AI 中心**（`/kb/{id}/ai`）
2. 点击 **重建索引**
3. 等待索引完成（系统会自动处理所有笔记文件）

> 首次索引后，后续修改笔记会自动更新索引。如果笔记大量变更，可以手动重建。

### 4.4 验证检索效果

在 **搜索** 页面（`/kb/{id}/search`）输入自然语言查询：

- _"上周的工作重点"_
- _"张三的客户跟进情况"_
- _"关于项目进度的笔记"_

如果能搜到相关内容，说明语义检索正常工作。

## 5. 配置 Embedding 模型

> Embedding 模型 = 语义检索的核心引擎。如果不配置，AI 只能做关键词匹配。

配置方式见第 4 节。配置完成后，在配置页可以看到语义检索状态：

| 状态 | 含义 |
|------|------|
| ✅ 已就绪（Ollama · bge-m3） | 本地 Ollama 正常工作 |
| ✅ 已就绪（硅基流动 · BAAI/bge-m3） | API 模式正常工作 |
| ❌ 不可用 | 需要检查配置 |

## 6. AI 对话

### 6.1 进入对话

1. 从左侧选择一个笔记库
2. 点击 **💬 对话**
3. 在输入框输入消息，按 Enter 发送

### 6.2 AI 能做什么

AI 自动判断你的意图并调用相应工具：

| 你说 | AI 会怎么做 |
|------|------------|
| _"帮我看看本周的日报"_ | `searchNotes("本周日报")` → 读取相关内容 → 总结 |
| _"记录一下今天完成了项目上线"_ | `writeFile` 写入笔记 + `addRecord` 记录数据集 |
| _"查一下待修复的 Bug"_ | `queryRecords("Bug", {"状态":"待修复"})` |
| _"帮我创建一个任务：下周完成测试"_ | `createTask("完成测试", "mid", "2026-07-05")` |
| _"切换到学习笔记库"_ | `switchKnowledgeBase("学习")` |
| _"帮我搜一下最新的 AI 新闻"_ | `webSearch("AI 最新新闻 2026")` |
| _"记住我叫张三"_ | `remember("user_profile", "称呼", "张三")` |

### 6.3 查看 AI 思考过程

每次 AI 回复后，回复上方会出现 **🤔 查看思考过程** 按钮，点击可展开 AI 的完整推理链：

- 理解用户意图
- 制定了什么计划
- 调用了哪些工具
- 每一步的结果

> 这个功能帮助你理解 AI 的决策过程，在 AI 回答不符合预期时可以快速定位问题。

### 6.4 切换模型

对话页面顶部有模型选择下拉框，可以在多个已配置的模型间切换。

### 6.5 清空对话

点击 **清空** 按钮可以清除当前笔记库的所有聊天记录。

## 7. 任务管理

### 7.1 创建任务

**方式一：AI 对话**

直接对 AI 说：
- _"帮我创建任务：完成季度报告，优先级高，截止日期7月15号"_
- _"记一个待办：买礼物"_
- _"添加任务：修复登录界面 Bug，高优先级"_

**方式二：任务页面**

1. 打开 **📋 任务与提醒** 页面
2. 点击 **新建任务**
3. 填写标题、描述、优先级、截止日期

### 7.2 管理任务

| 操作 | AI 对话 | 页面操作 |
|------|---------|---------|
| 查看任务 | _"我的任务有哪些"_ | 任务列表页 |
| 完成任务 | _"完成登录Bug那个任务"_ | 点击 ✅ |
| 更新任务 | _"把测试任务的优先级改为高"_ | 点击编辑 |
| 删除任务 | _"删除购买服务器那个任务"_ | 点击删除 |

## 8. 提醒管理

### 8.1 创建提醒

**方式一：AI 对话**

直接对 AI 说：
- _"提醒我每天下午6点写日报"_ — 创建每天提醒
- _"提醒我明天上午10点开会"_ — 创建一次性提醒
- _"每周一早上9点提醒我开周会"_ — 创建每周提醒

**方式二：提醒页面**

1. 打开 **📋 任务与提醒** → **提醒** 标签
2. 点击 **新建提醒**
3. 填写名称、消息、类型、时间

### 8.2 提醒类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `daily` | 每天固定时间触发 | 每天 18:00 写日报 |
| `once` | 指定日期触发一次 | 明天 10:00 开会 |
| `weekly` | 每周固定星期几触发 | 每周一 09:00 开周会 |
| `monthly` | 每月固定日期触发 | 每月 1 号 09:00 做月报 |

### 8.3 管理提醒

| 操作 | AI 对话 | 页面操作 |
|------|---------|---------|
| 查看提醒 | _"我的提醒有哪些"_ | 提醒列表 |
| 开关提醒 | _"关闭日报提醒"_ | 点击开关 |
| 修改提醒 | _"把日报提醒改为下午5点"_ | 点击编辑 |
| 删除提醒 | _"删除开会提醒"_ | 点击删除 |

> 提醒触发时会通过飞书推送消息（如果配置了飞书）。

## 9. 数据中心（数据集）

数据集是**结构化数据**的管理系统，用于记录客户、Bug、项目等有固定格式的数据。

### 9.1 创建数据集

1. 打开 **📦 数据中心**
2. 点击 **新建数据集**
3. 填写名称和描述
4. 定义字段（Schema）：

| 字段设置 | 说明 |
|---------|------|
| 字段名 | 如：客户名、状态、优先级 |
| 字段类型 | text / number / select（下拉选项） |
| 下拉选项 | 如果类型是 select，填写可选值，用逗号分隔 |

示例：客户跟进数据集

| 字段名 | 类型 | 选项 |
|--------|------|------|
| 公司 | text | — |
| 联系人 | text | — |
| 阶段 | select | 初步接触,需求沟通,方案报价,签约,售后 |
| 跟进时间 | text | — |

### 9.2 添加数据

**方式一：AI 对话**

_"记录一个客户：ABC 公司，联系人张三，阶段是方案报价"_

**方式二：页面操作**

在数据集详情页点击 **新增** 填写数据。

### 9.3 查询数据

**方式一：AI 对话**

- _"查一下报价阶段的客户"_
- _"本周新增了哪些 Bug"_
- _"高优先级的待修复问题有哪些"_

**方式二：页面操作**

在数据集详情页浏览、搜索、筛选。

### 9.4 导入数据

支持从 JSON 或 Excel 文件批量导入数据到数据集。

## 10. AI 引导与记忆

### 10.1 AGENTS.md — 自定义 AI 行为规则

在每个笔记库的根目录创建 `AGENTS.md` 文件，AI 会自动读取并按照其中的规则行事。

示例 `AGENTS.md`：

```markdown
## 行为规则
- 在回复中必须引用来源文件
- 客户记录优先使用 logRecord 工具
- 日报格式：日期 + 任务 + 进展 + 明日计划
```

编辑方式：
1. 打开 **AI 中心**（`/kb/{id}/ai`）
2. 切换到 **AGENTS.md** 标签
3. 编辑内容并保存

### 10.2 AI 记忆

AI 会记住你在对话中透露的个人信息（称呼、偏好、项目等），并在后续对话中使用。

**AI 能记住什么：**

| 分类 | 内容 | 示例 |
|------|------|------|
| 用户画像 | 名字、职业、身份 | 用户叫老齐 |
| 偏好设置 | 语言风格、习惯 | 喜欢简洁回复 |
| 项目信息 | 项目名、技术栈 | 项目用 Spring Boot |
| 关键事实 | 重要约定、日期 | 每周五有周会 |
| 目标跟踪 | 当前目标、进展 | 正在开发 v2.0 |

**管理记忆：**

- AI 在对话中会自动存储关键信息（异步）
- 你也可以主动让 AI 记住："记住我叫张三"、"记住我在做 Java 项目"
- 目前不能手动编辑记忆列表

## 11. 图片识别

支持场景：
- 识别截图中的文字
- 识别照片中的物体

## 13. 飞书集成

### 13.1 Webhook 推送

1. 在飞书群中添加 Webhook 机器人
2. 复制 Webhook URL
3. 在配置页填入 URL
4. 点击 **测试** 验证

5. 在飞书群中 @机器人 即可对话

## 14. 定时日报

### 14.1 开启日报

默认每个笔记库开启自动日报，每天 **09:00** 自动生成。

管理方式：
- 在笔记库设置中开启/关闭自动日报
- 在 **AI 中心** 中点击 **生成日报** 可手动触发

### 14.2 日报内容

日报由 AI 自动生成，包含：
1. 从笔记库读取今日工作记录
2. 从数据集查询任务/Bug/项目状态
3. 结合两者生成综合日报
4. 保存到笔记库

日报保存位置：`{笔记库目录}/AI/日报/`

## 15. 常见问题

### Q: 启动后页面空白 / 无法访问

- 确认端口 6790 未被占用：`lsof -i :6790`
- 确认 Java 版本 ≥ 17：`java -version`
- 查看日志：`tail -f app.log`

### Q: AI 回复 "LLM API Key 未配置"

- 打开配置页检查 API Key 是否已填写
- 确认 Base URL 正确（不要以 /v1 结尾）
- 保存后刷新页面重试

### Q: AI 回复 "Embedding 服务不可用"

- 语义搜索降级为文件名搜索，不影响对话
- 如需语义搜索，配置 Ollama 或在线 Embedding API
- 配置后重启系统

### Q: 语义搜索搜不到内容

- 确认已执行 **重建索引**
- 确认笔记文件是支持的格式（.md / .json / .txt）
- 在搜索页面测试不同的关键词
- 检查 Embedding 服务状态（配置页有状态显示）

### Q: 笔记库路径设置后不生效

- 路径必须是已存在的绝对路径
- 检查运行 Java 的用户是否有该目录的读取权限
- Windows 用户使用正斜杠：`C:/Users/me/notes`

### Q: AI 调用工具时出错

- 查看 AI 回复上方的"思考过程"，定位哪一步出错
- 笔记操作错误：检查文件路径和权限
- 数据集操作错误：检查数据集是否存在
- API 错误：检查 API Key 余额

### Q: 飞书收不到消息

- 确认 Webhook URL 正确
- 点击 **测试** 验证连接
- 检查飞书群是否开启了机器人

### Q: 如何备份数据

- 笔记文件：备份笔记库目录
- 数据库：备份 `memory.db` 文件
- 配置文件：备份 `config.json`
