## 一、核心思路

v1.1 给了干净的结构化数据（笔记库 JSON），v1.2 给了 AgentAnalysisService 自主读文件、分析、输出 Markdown 的能力。
v1.3 是在这之上套 5 个**预设场景模板**，让 AI 的输出更可控、可用、可落地。

```
底层复用（不重复造轮子）：
  LlmService（已有）              → AI 对话
  OpenAiChatModel / DeepSeekModel（已有）→ AI 引擎
  poi-ooxml（pom.xml 已有）       → Java 原生写 .pptx
  commonmark（pom.xml 已有）      → Java 解析 Markdown
  pandoc（系统已装）              → md 转 docx / PDF
  NoteDataScanner（v1.1 新增）   → 扫笔记库拿数据
  DataIngressService（v1.1 新增）→ 写笔记库

本次新增（上层 5 个场景）：
  OfficeSceneController       → 统一入口 Tab
  ImageGenService             → 图片生成（抽象接口 + OpenAI 实现）
  MarkdownEditor（前端）       → 双向润色
  PPT 预览器（前端）           → md → 幻灯片
  ExportService               → md → .pptx / .docx / .pdf
```

## 三、五个子场景详细设计

### 3.1 写作

```
输入： 模板（文章/邮件/博客/周报/月报/感谢信/述职/演讲稿/自荐信）
       主题（自然语言描述）
       可选：让 AI 参考笔记库哪些模块的数据（默认 AI 自主选）

AI Prompt：（模板化 System Prompt）
  "你是一个资深{模板类型}写作者。
   主题：{主题}
   知识库今日相关文件（AI 自主 @Tool 可读取）：
     {listDir / searchFiles 结果}
   请用 Markdown 写一篇完整的{模板类型}。
   风格：{正式/轻松/专业/口语}
   字数：{500/1000/2000}"

输出：  Markdown（自动保存到笔记库所选位置）
       可选导出：.docx（pandoc）
```

模板清单（可扩展，存在笔记库 AI/写作/模板/ 下，用户自己也能加）：

```
模板名称        System Prompt 核心
文章           结构化 Markdown，有标题/引言/正文/总结
邮件           有称呼/正文/结尾/签名，段落短
博客           口语化，段落松，有小标题，适合手机阅读
周报           按日期分组：本周完成 / 进行中 / 下周计划 / 问题风险
月报           同上，有月度数据对比
感谢信         真诚，具体事件，具体感谢点
述职报告       业绩数字 + 个人贡献 + 成长 + 下一步
演讲稿         开场白吸引人 + 3 个要点 + 结尾呼应
自荐信         自我介绍 + 匹配点 + 动机 + 结尾
```

### 3.2 分析报告

```
输入：  主题（自然语言，如"6月客户流失分析"、"CSDN AI文章趋势"）
       数据源（两种模式）：
         A. AI 自主  → AgentAnalysisService 扫笔记库，自己决定
         B. 用户指定  → 勾选模块/数据集（客户/data/项目/优合/AI/数据/CSDN文章）

AI Prompt：（在 v1.2 AgentAnalysisService 的 Prompt 上加数据约束）
  "你是一个资深数据分析师。
   主题：{主题}
   数据源：{数据源列表或'AI 自主选'}
   请自主 readFile 上面数据源里你需要的文件。
   输出结构：
     ---frontmatter（日期/知识库/涉及文件列表）---
     1. 执行摘要（3 句话）
     2. 数据看板（关键数字表格/列表）
     3. 分析正文（分 2-4 小节）
     4. 趋势对比（如果有昨日/上月数据）
     5. 结论与建议（1-3 条具体可执行）"

输出：  Markdown（带 frontmatter，可被日报扫描器识别）
       可选导出：.docx / .pdf（pandoc）
```

### 3.3 PPT

```
输入：  模板（产品介绍 / 汇报 / 培训 / 年终总结 / 方案演示 / 演讲）
       主题
       大纲（用户可选手写大纲，或让 AI 自主拟大纲）

流程（3 步）：
  Step 1. AI 生成大纲 → 展示给用户确认/修改
  Step 2. 按大纲逐页生成
         "你是一个资深演示文稿专家。
          主题：{主题}
          当前页：{当前页标题}
          大纲上下文：{前面几页 + 后面几页}
          请用 Markdown 写这一页。
          规则：
            - 第一行 = 页面标题（# 开头）
            - 要点控制在 3-5 条
            - 可以用 --- 分隔内容块
            - 不要写废话，每页一个核心思想
            - 可以嵌入 ASCII 表格"
  Step 3. 预览 & 导出

输出：
  a) Markdown（每页一个 # 标题，--- 分隔）
  b) 前端内置预览器（reveal.js CDN 引入，零安装）
  c) Java 后端 poi-ooxml 原生生成 .pptx（可选）

PPT 预览器实现：
  reveal.js（CDN 引入）
  ```
  <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/reveal.js/5.0.0/reveal.min.css">
  ```
  后端把 AI 生成的 Markdown 按 `^# ` 分割成页，
  每页包 `<section>`，丢给 reveal.js 渲染。

PPT 导出 .pptx（poi-ooxml 已有，Java 原生写）：
  ```java
  public byte[] exportPptx(List<Slide> slides) {
      XMLSlideShow ppt = new XMLSlideShow();
      for (Slide s : slides) {
          XSLFSlide slide = ppt.createSlide();
          // 标题：大字加粗
          // 要点：XSLFTextBox + XSLFRichTextString
          // 表格：XSLFTable
      }
      ByteArrayOutputStream out = new ByteArrayOutputStream();
      ppt.write(out);
      return out.toByteArray();
  }
  ```
  前端：[📤 下载 .pptx]

```

### 3.4 工作文档

```
输入：  模板（合同 / 产品说明书 / 需求文档 / 方案 / 规章制度 / 备忘录 / 会议纪要）
       变量参数（模板里留 {{客户名}}、{{日期}}、{{金额}} 占位符，AI 填充）
       可选：让 AI 参考笔记库相关模块

AI Prompt：
  "你是一个资深{模板类型}撰写人。
   模板：{模板类型}
   变量：{客户名: 启航电商, 日期: 2026-06-20, ...}
   知识库相关文件（AI 自主 @Tool）：{...}
   输出严格符合{模板类型}规范。"

输出：  Markdown
       可选导出：.docx（pandoc，对 Word 排版更友好）
                .pdf（pandoc + wkhtmltopdf 或直接 md → html → print PDF）

pandoc 调用（Java ProcessBuilder）：
  ```java
  public byte[] exportDocx(String markdown) throws IOException, InterruptedException {
      Process p = new ProcessBuilder(
          "pandoc", "-f", "markdown", "-t", "docx", "-o", "-"
      ).start();
      try (OutputStream os = p.getOutputStream()) {
          os.write(markdown.getBytes());
      }
      return p.getInputStream().readAllBytes();
  }
  ```
```

### 3.5 生成图片

```
输入：  图片描述（自然语言）
       风格（写实 / 插画 / 扁平化 / 3D / 手绘 / 像素风 / 照片风）
       尺寸（1024x1024 / 1536x1024（横图）/ 1024x1536（竖图））
       用途（配图 / 封面 / 流程图 / 产品图 / 头像）→ 辅助 Prompt 优化

后端：ImageGenService（抽象接口 + OpenAI 实现）
  ```java
  public interface ImageGenService {
      /**
       * 生成图片，返回图片 URL 或 base64
       */
      GeneratedImage generate(String prompt, String style, Size size);

      record GeneratedImage(String url, String base64, String revisedPrompt) {}
  }

  @Service
  public class OpenAiImageGenService implements ImageGenService {
      // 用 Spring AI 的 ImageModel 接口（OpenAiImageModel）
      // 或直接 REST 调 OpenAI images generations API（dall-e-3）
  }
  ```

Spring AI openai starter 支持图片生成：
  - Spring AI 1.0-M4+ 有 `ImageModel` 接口
  - OpenAI 实现支持 dall-e-3（默认）和 dall-e-2
  - 不需要额外依赖，pom.xml 已引入 spring-ai-starter-model-openai

保存：
  生成的图片保存到笔记库所选位置（保存文件对话框）
  默认：知识库/AI/图片/{用途}/YYYY-MM-DD-简短描述.png

输出：  .png（原图存笔记库）
       Markdown 预览块（![描述](路径)）

Prompt 优化（风格 + 用途 自动拼接）：
  ```
  原始描述："启航电商 ERP 产品主图"
  优化后 prompt 给 AI：
    "Product photo style, 启航电商 ERP product shot,
     professional lighting, clean background, horizontal 16:9,
     suitable for PPT cover"
  （style 和用途由后端模板拼接，用户原始描述作为核心）
  ```

## 四、双向润色编辑器

所有 5 个子场景生成 Markdown 后，都进入"编辑模式"：

```
┌─────────────────────────────────────────────────────┐
│  📝 写作 · 编辑中                                    │
│                                                     │
│  [AI 生成]  [重新生成]  [💾 保存]  [📤 导出 Word]    │
│                                                     │
│  ┌─────────────────────────┐  ┌───────────────────┐ │
│  │ 编辑区（textarea）       │  │ 预览区（commonmark)│ │
│  │                         │  │                   │ │
│  │ 选中一段文字（蓝色背景） │  │   实时同步预览    │ │
│  │  ┌───────────────────┐  │  │                   │ │
│  │  │润色│续写│翻译│改色│  │  │                   │ │
│  │  │扩写│缩写│重写│    │  │  │                   │ │
│  │  └───────────────────┘  │  │                   │ │
│  └─────────────────────────┘  └───────────────────┘ │
│                                                     │
│  知识库：[工作 ▾]  保存到：[写作/周报/2026-06-20.md]│
└─────────────────────────────────────────────────────┘
```

实现：
```
前端：
  textarea 编辑 + commonmark.js CDN 渲染右侧预览（已支持 GFM tables）
  window.getSelection() 拿到选中文字
  悬浮菜单：6 个按钮，每个对应一个 AI 指令

后端：
  @PostMapping("/api/office/polish")
  请求体 { fullText, selectedText, action, targetLang(可选) }
  Prompt：
    润色："润色这段文字，保持原意，更专业流畅：{selected}"
    续写："续写这段文字，延续写作风格：{selected}"
    翻译："翻译成{targetLang}：{selected}"
    改语气："改成{正式/轻松/专业/口语}语气：{selected}"
    扩写："扩写 2 倍，保持结构：{selected}"
    缩写："缩写到 1/3，保留核心：{selected}"
    重写："用不同方式重写：{selected}"
  返回替换后的文字
  前端：把选中文字替换成 AI 返回结果 + 自动选中新文字
```

## 五、后端 API 设计

```
OfficeSceneController（统一入口）：

  GET  /office                    → 渲染办公场景 Tab 页面（模板选择视图）
  GET  /office/{scene}            → 渲染具体子场景页面
                                    scene = writing | report | ppt | doc | image
  POST /api/office/{scene}/generate
    写作 { template, topic, kbId, style, length }
    报告 { topic, kbId, dataMode("auto"|"manual"), sources[] }
    PPT  { template, topic, outline?(null=AI拟), kbId }
    文档 { template, variables{}, kbId }
    图片 { prompt, style, size, usage, kbId }
    返回 { content, filePath?, preview? }

  POST /api/office/polish         → 润色/续写/翻译/改语气/扩写/缩写/重写
    { fullText, selected, action, targetLang? }
    返回 { replaced, fullText }

  GET  /api/office/export/{format} → 导出
    format = docx | pptx | pdf | markdown
    参数 filePath（笔记库中的相对路径）
    返回 文件流（Content-Disposition: attachment）

  GET  /api/office/templates     → 列出所有模板
    返回 [{id, name, scene, prompt}]

  POST /api/office/image/save    → 生成图片保存到笔记库
    { imageBytesBase64, kbId, targetPath, fileName }
```

ImageGenService：
```java
public interface ImageGenService {
    GeneratedImage generate(String prompt, String style, Size size) throws Exception;
    record GeneratedImage(String localPath, String revisedPrompt, int width, int height) {}
}
// 默认实现：OpenAI DALL-E 3（Spring AI ImageModel）
// 可替换为其他多模态大模型图片生成 API
```

ExportService：
```java
@Service
public class ExportService {
    // md → docx（pandoc）
    public byte[] toDocx(String markdown) throws IOException, InterruptedException;
    // md → pptx（poi-ooxml，Java 原生，无需 pandoc）
    public byte[] toPptx(List<Slide> slides) throws IOException;
    // md → pdf（pandoc）
    public byte[] toPdf(String markdown) throws IOException, InterruptedException;
    // md → html（commonmark，前端已有的解析器）
    public String toHtml(String markdown);
}
```
