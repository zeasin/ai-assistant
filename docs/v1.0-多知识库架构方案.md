# v1.0 重大升级：多知识库架构

> 版本：1.0
> 状态：方案确认，待实施
> 日期：2026-06-18

---

## 一、概述

从"单知识库"升级为"多知识库"架构，用户可在同一实例中管理多个独立的知识库（如工作、学习），数据、对话、记忆、模块完全隔离，共享 AI 引擎和全局配置。

### 2.2 交互规则

| 操作 | 行为 |
|------|------|
| 点击 KB 名称（如"工作"） | 跳转 `/kb/{id}` 概览页，localStorage 写入 `kbId` |
| 点击 KB 二级菜单 | 跳转 `/kb/{id}/{page}`，写入 `kbId` |
| 点击全局页面（首页/对话/识图/配置/日志/帮助） | 读取 `localStorage.kbId`，展示对应 KB 的数据 |
| 首次访问/无 KB 时 | 显示 KB 选择门户 |
| 刷新页面 | localStorage 保持当前 KB |

### 2.3 KB 内部菜单名可配置

每个知识库的二级菜单名称通过 `labels` 字段（JSON map）自定义：

```json
// 工作 KB 的 labels
{"tasks": "任务", "reminders": "提醒", "modules": "模块", "notes": "笔记", "config": "配置"}

// 学习 KB 的 labels
{"tasks": "学习计划", "reminders": "复习提醒", "modules": "科目", "notes": "学习笔记", "config": "配置"}
```

代码层面，任务使用同一套 `TaskService`，提醒使用同一套 `ReminderService`，仅显示名称不同。

## 三、数据模型

### 3.1 新增表

```sql
CREATE TABLE knowledge_bases (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL,                     -- "工作" / "学习"
    notes_dir   TEXT NOT NULL,                     -- 笔记库绝对路径
    labels      TEXT NOT NULL DEFAULT '{}',        -- 菜单名配置（JSON map）
    is_active   INTEGER NOT NULL DEFAULT 0,
    sort_order  INTEGER NOT NULL DEFAULT 0,
    created_at  TEXT NOT NULL
);
```

### 3.2 修改表

**modules 表新增 kb_id：**
```sql
ALTER TABLE modules ADD COLUMN kb_id INTEGER NOT NULL DEFAULT 1;
CREATE INDEX idx_modules_kb ON modules(kb_id);
```
启动时将现有模块全部设 `kb_id=1`（第一个 KB）。

**sessions 表新增 kb_id：**
```sql
ALTER TABLE sessions ADD COLUMN kb_id INTEGER NOT NULL DEFAULT 1;
CREATE INDEX idx_sessions_kb ON sessions(kb_id);
```

### 3.3 不变的表

| 表 | 原因 |
|----|------|
| `messages` | 通过 `sessions.kb_id` 关联，无需加列 |
| `turn_embeddings` | 通过 `sessions.kb_id` 关联 |
| `llm_profiles` | 全局共享，与 KB 无关 |
| `coding_records` | 编程 AI 独立功能，不与 KB 关联 |

### 3.4 JSON 文件自动按 KB 隔离

所有基于 notesDir 的 JSON 文件，通过 `ConfigService.getNotesDir()` 切换路径后天然隔离：

| 文件 | 原路径 | 新路径 |
|------|--------|--------|
| 任务 | `{notesDir}/AI/任务/data.json` | `{activeKb.notesDir}/AI/任务/data.json` |
| 提醒 | `{notesDir}/AI/提醒/reminders.json` | `{activeKb.notesDir}/AI/提醒/reminders.json` |
| 日报 | `{notesDir}/AI/综合日报/` | `{activeKb.notesDir}/AI/综合日报/` |
| 记忆 | `{notesDir}/AI/记忆/` | `{activeKb.notesDir}/AI/记忆/` |
| 采集器 | `{notesDir}/AI/采集器/` | `{activeKb.notesDir}/AI/采集器/` |
| AGENTS.md | `{notesDir}/AGENTS.md` | `{activeKb.notesDir}/AGENTS.md` |

## 七、启动迁移流程

首次启动 v1.0 版本时：

1. 创建 `knowledge_bases` 表
2. 读取 `config.json` 的 `notesDir` 字段，自动创建第一条 KB（"工作"）
3. 为 `modules` 表加 `kb_id` 列，现有模块全部设 `kb_id=1`
4. 为 `sessions` 表加 `kb_id` 列，现有 session 全部设 `kb_id=1`
5. 将 `config.json` 中的 `notesDir` 标记为旧配置，迁移完成后不再读取（优先使用 `knowledge_bases` 表）

## 八、隔离保障

| 维度 | 隔离机制 |
|------|---------|
| 聊天历史 | `sessions.kb_id` 过滤，向量召回限定 `kb_id` |
| 模块/科目 | `modules.kb_id` 过滤，UI 只显示当前 KB 的模块 |
| AI 记忆 | 走 `{activeKb.notesDir}/AI/记忆/` |
| AGENTS.md | 走 `{activeKb.notesDir}/AGENTS.md` |
| 文件操作（NoteTools） | `baseDir` = 当前 KB 的 notesDir |
| 任务数据 | `{activeKb.notesDir}/AI/任务/data.json` |
| 提醒数据 | `{activeKb.notesDir}/AI/提醒/reminders.json` |
| 日报 | `{activeKb.notesDir}/AI/综合日报/` |
| 识图结果 | 保存到当前 KB 的 notesDir 下 |
| 笔记浏览 | 遍历当前 KB 的 notesDir |
