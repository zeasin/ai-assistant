# 飞书机器人配置指南

本文档介绍如何配置飞书机器人，实现 AI 对话功能。

## 一、飞书开放平台配置

### 1. 创建应用

1. 访问 [飞书开放平台](https://open.feishu.cn/app)
2. 点击「创建企业自建应用」
3. 填写应用名称和描述
4. 获取 `App ID` 和 `App Secret`

### 2. 添加应用能力

1. 进入应用详情
2. 点击「添加应用能力」
3. 选择「机器人」并添加

### 3. 配置权限

1. 进入「权限管理」
2. 添加以下权限：
   - `im:message:send_as_bot` - 发送消息
   - `im:message` - 获取与发送单聊、群组消息

### 4. 配置事件订阅

1. 进入「事件与回调」
2. 点击「添加事件」
3. 搜索并添加：`im.message.receive_v1`（接收消息）

### 5. 发布应用

1. 点击「版本管理与发布」
2. 创建版本并提交审核（如需要）
3. 发布应用

## 二、AI 助理配置

### 1. 基础配置

在配置页面填写：

| 配置项 | 说明 |
|--------|------|
| App ID | 飞书应用的 App ID |
| App Secret | 飞书应用的 App Secret |
| Chat ID | 接收消息的群 ID 或会话 ID |
| 启用消息轮询 | 勾选以启用长连接 |

### 2. 消息格式

消息内容为 JSON 格式，示例：

```json
{
  "text": "你好，请问今天天气怎么样？"
}
```

### 3. 回复机制

- **私聊(p2p)**：直接回复用户消息
- **群聊**：在群内回复用户消息

## 三、验证配置

### 1. 启动应用

```bash
mvn package -q
java -jar target/assistant-v2-2.0.0.jar
```

### 2. 检查日志

启动成功后，应看到以下日志：

```
[飞书长连接] 正在启动...
[飞书长连接] ✅ 启动成功，等待消息...
```

### 3. 测试

1. 在飞书找到你的机器人
2. 发送一条消息
3. 查看日志确认收到消息：
   ```
   [飞书长连接] 收到消息: msgType=text
   [飞书长连接] ✅ 收到用户消息: 你好
   ```
4. 确认收到 AI 回复

## 四、常见问题

### 1. 长连接启动失败

**症状**：日志显示启动失败或异常

**排查**：
- 确认 App ID 和 App Secret 正确
- 确认应用已发布
- 检查网络连接

### 2. 收不到消息

**排查**：
- 确认已添加「机器人」能力
- 确认已订阅 `im.message.receive_v1` 事件
- 确认应用已发布并对该用户/群可见
- 检查日志是否有 `[飞书长连接] 收到消息` 输出

### 3. 发送消息失败

**排查**：
- 确认已添加 `im:message:send_as_bot` 权限
- 确认应用已发布

### 4. AI 无回复

**排查**：
- 确认 opencode serve 已启动
- 检查日志是否有 `[飞书长连接] AI 回复长度` 输出
- 确认 AI 服务正常运行

## 五、技术架构

```
用户发送消息 → 飞书服务器 → WebSocket长连接 → AI助理
                    ↓
             AI助理 → opencode AI → 回复内容
                    ↓
             飞书服务器 → 用户收到回复
```

**核心组件**：
- `FeishuLongConnectionService`：管理 WebSocket 长连接
- `OpenCodeService`：调用 AI 服务
- `FeishuService`：发送飞书消息
