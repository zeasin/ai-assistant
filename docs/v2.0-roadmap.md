## AI 能力增强

### Plan → Execute → Reflect 显式循环

当前 v1.0 的工具编排是**隐式的**——LLM 内部自动决定调用哪个工具，用户只看到最终结果。

v2.0 计划增加**显式的**多步任务分解能力：

| | 隐式（v1.0） | 显式（v2.0） |
|---|---|---|
| **做什么** | LLM 内部决定调哪个工具 | 先输出计划，逐步执行，逐步反思 |
| **作用层** | 单步执行层 | 任务编排层 |
| **可见性** | 用户只看到最终结果 | 用户能看到计划和每步进展 |
| **适合场景** | 简单请求（读个文件、搜个笔记） | 复杂多步任务（整理+提取+创建+提醒） |

显式编排**不会替代**隐式工具调用，而是作为上层循环，按需触发：

```
用户: "整理本周会议纪要，提取行动项，创建任务"

[Plan]    → AI 输出: 步骤1/2/3
[Execute] → 步骤1: 调用 readFile (隐式工具编排)
           → 步骤2: 调用 writeFile
           → 步骤3: 调用 writeFile
[Reflect] → 每步后评估是否达成，必要时调整
```

**已验证的技术方案**：
- `AgentOrchestrator` 负责外层 Plan/Execute/Reflect 循环
- Spring AI `ChatClient` + `defaultTools()` 负责内层工具执行
- 通过 `needsPlan()` 自动判断简单/复杂任务，按需切换

**待验证**：
- 多轮 LLM 调用的 token 消耗和延迟是否可接受
- reflect/replan 的 prompt 调优

### 插件化工具

当前工具（NoteTools）是编译时固定的。v2.0 计划支持运行时动态加载工具：

- 工具以 JAR 包形式提供，放到指定目录
- 启动时或运行时扫描加载
- `ToolRegistry` 扩展为支持动态注册/卸载
- 不同知识库可加载不同工具集

---

### 多 Agent 协作

当前是单 Agent 模式（NoteAssistantService 一个 AI 处理所有请求）。v2.0 考虑：

- 专用 Agent（如数据分析 Agent、代码排查 Agent）
- Agent 之间通过消息传递协作
- 主 Agent 负责任务分发

## 办公场景应用

补齐日常办公场景的 AI 能力：

| 功能 | 说明 |
|------|------|
| 总结报告 | 基于笔记/数据自动生成结构化总结报告，支持多种模板 |
| 生成 PPT | AI 根据内容自动生成演示文稿（Markdown → PPTX） |
| 生成图片 | AI 生成配图、流程图、思维导图等可视化素材 |
| 导出文档 | 将笔记/报告导出为 PDF、Word、HTML 等格式 |

**技术方向**：
- PPT 生成：基于 `poi-ooxml` 或 `python-pptx`（通过 HTTP 调用）
- 图片生成：接入 AI 图片生成 API（如 DALL-E、Stable Diffusion）
- 文档导出：`poi`（Word）、`openpdf`（PDF）
