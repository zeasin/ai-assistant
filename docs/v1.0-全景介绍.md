# AI 助理 v1.0 全景介绍

> 自带 AI 大脑，零外部依赖。多知识库，一步运行。

---

## 一、系统定位

AI 助理是一个**完全自包含的本地 AI 应用**——基于 Spring Boot 4.1 + Spring AI 2.0 构建，原生集成多知识库管理、AI 对话（LLM 推理 + `@Tool` 工具编排 + 语义向量检索）、编程 AI（Pi CLI 代码自动排查）、多模态图片识别。

核心理念：**AI 是大脑本身，不是桥梁**。用户只需提供数据（笔记库目录）和规则（AGENTS.md），AI 即可自主理解、分析、操作这些数据。

### 2.3 AI 调用链路

```
用户输入
  ↓
ChatController.send() (SSE 流式)
  ↓
NoteAssistantService.streamChat()
  ├── SessionService.buildHistoryContext()  ← 语义召回 + 时间窗口
  ├── NoteTools.setCurrentKbId(kbId)        ← ThreadLocal 隔离
  ├── ChatClient.builder(chatModel)
  │     .defaultTools(toolRegistry.getToolArray())  ← @Tool 自动注入
  │     .build()
  └── client.prompt().system(SYSTEM_PROMPT).user(fullMessage).stream()
        ↓
      Spring AI ToolCallingAdvisor 自动判断是否调用工具
        ├── readFile(path)
        ├── writeFile(path, content)
        ├── listDir(path)
        └── searchFiles(keyword)
        ↓
      工具执行 → 结果回传 LLM → 最终回复
```

### 2.4 语义检索流程

```
用户提问
  ↓
Ollama embed(text) → 768 维向量
  ↓
SQLite turn_embeddings 表（按 kb_id 隔离）
  ↓
余弦相似度排序（threshold ≥ 0.5）
  ↓
取 Top 8 轮（每会话最多 4 轮，全局最多 5 轮）
  ↓
注入为历史上下文给 AI
```

向量嵌入支持两种模式：
- **Ollama 本地**：`nomic-embed-text` 模型，完全离线
- **OpenAI 兼容 API**：如硅基流动等云服务，通过 `embeddingApiKey` 配置

## 三、功能全景

### 3.1 导航结构

```
[AI 助理]  首页  对话  识图  数据中心  计划  笔记库▾  编程AI  配置  日志  帮助

                       ┌────────────┼────────────┐
                       ▼            ▼            ▼
                    [工作]       [学习]        [自定义]
                    ├─ 📋 任务   ├─ 📋 学习计划  ├─ 各自二级菜单
                    ├─ 🔔 提醒   ├─ 🔔 复习提醒  │
                    ├─ 🤖 模块   ├─ 🤖 科目     │
                    ├─ 📄 笔记   ├─ 📄 笔记     │
                    └─ ⚙️ 配置   └─ ⚙️ 配置     │
```

### 3.2 核心功能详解

#### 🏠 首页（`/`）

- 全局状态展示
- 多知识库摘要看板：按当前 KB 展示日报、待办、任务数
- 每日 09:00 自动生成综合日报

#### 💬 AI 对话（`/chat`）

- **连续对话**：去掉多会话管理，按 KB 展示连续聊天记录
- **SSE 流式输出**：实时显示 AI 回复
- **工具编排**：AI 通过 `@Tool` 自动读写笔记库文件
- **语义记忆**：跨会话向量检索，自动召回相关历史上下文
- **对话导出**：支持导出为 Markdown 保存至笔记库

#### 🗂️ 知识库系统（`/kb/{id}`）

v1.0 核心升级——一个实例同时管理多个知识库，完全隔离。

**隔离维度**：

| 维度 | 隔离方式 |
|------|---------|
| 聊天历史 | `sessions.kb_id` + 向量召回限定 `kb_id` |
| 模块/科目 | `modules.kb_id` 过滤 |
| AI 记忆 | `{notesDir}/AI/记忆/` 隔离 |
| 任务/提醒 | SQLite 按 `kb_id` 隔离 |
| 文件操作 | NoteTools 路径基于当前 KB |

**菜单名可配置**：每个 KB 可自定义二级菜单名称（如"任务"→"学习计划"）。

#### 🤖 AI 工具编排（`@Tool`）

AI 通过 Spring AI 的 `ToolCallingAdvisor` 自动判断何时调用工具，无需手写 ReAct 循环。

**工具自注册**：`ToolRegistry` 统一管理所有工具集，新增工具只需写 `@Component` + `@Tool` 方法，注册到 `ToolRegistry` 即可。

| 工具 | 功能 | 触发场景 |
|------|------|---------|
| `readFile` | 读取笔记库文件内容 | AI 需要了解数据格式或历史内容 |
| `writeFile` | 写入文件（先读后写，防覆盖） | 创建/更新 JSON 记录、Markdown 文档 |
| `listDir` | 列出目录内容 | AI 需要探索笔记库结构 |
| `searchFiles` | 按文件名关键词搜索 | 查找特定客户、BUG、日报等 |

**System Prompt 核心逻辑**：
1. 第一步：读取 `AGENTS.md` 了解笔记库规则（目录结构、字段格式、编号规则）
2. 第二步：按规则执行操作（搜索→读取→生成→写入）
3. 硬性约束：每次必须重新读取规则，不假设路径、不猜格式

**典型场景**：

| 用户说 | AI 执行 | 结果 |
|--------|---------|------|
| "查张三的跟进记录" | `searchFiles("张三")` → `readFile(...)` → 总结 | 文字回复 |
| "记录：拜访 ABC 公司" | 读取规则 → 读取现有数据 → `writeFile(...)` | 确认已写入 |
| "张三改为已签约" | 读取数据 → 修改 → `writeFile(...)` | 确认已更新 |
| "生成本周周报" | 读取多文件 → 汇总 → AI 撰写 → `writeFile(...)` | 保存+回复 |

#### 📋 任务/学习计划（`/planner`）

轻量级任务看板，按知识库隔离，SQLite 存储。

**功能**：
- 任务创建（标题、描述、优先级、截止日期）
- 状态管理（看板视图）
- Web UI 操作

**数据存储**：SQLite `tasks` 表 + 笔记库 JSON 双写

#### ⏰ 提醒/复习提醒（`/planner`）

基于时间的动态提醒系统，按知识库隔离。

**提醒类型**：
- `daily`：每天定时
- 自定义 Cron 表达式
- 触发时通过飞书推送

**定时检查**：`SchedulerService` 每分钟检查到期提醒

**数据存储**：SQLite `reminders` 表 + 笔记库 JSON 双写

#### 🧩 模块/科目系统（`/kb/{id}/modules`）

每个知识库独立管理自己的模块，每个模块对应知识库中的一个目录。

**模块定义**：
- `id`：唯一标识
- `name`：显示名称（如"项目"、"数学"）
- `dir`：知识库中的相对路径
- `icon`：图标
- `prompt`：模块专属 AI 提示词
- `dataFiles`：关联的数据文件列表

**模块详情页**（`/kb/{id}/notes?dir=xxx`）：
- **文件 Tab**：文件 CRUD（新建/编辑/重命名/删除）
- **AI 分析 Tab**：Agent 自主探索目录并生成分析报告
- **原始数据 Tab**：JSON 数据表格化展示 + CRUD + 列设置

#### 🖼️ 图片识别（`/image`）

基于多模态 Vision API 的图片分析功能，按知识库隔离。

**功能**：
- 自动扫描当前 KB 笔记库中的图片（PNG/JPG/GIF/BMP/WebP）
- 缩略图浏览
- AI 多模态分析
- 分析结果保存到当前 KB

**技术实现**：`LlmService.chatWithImage()` 支持 data URI 传输，使用 OpenAI 兼容协议调用多模态模型。

#### 📦 数据中心（`/datacenter`）

集中管理结构化数据集，完全 SQLite 存储。

**功能**：
- 创建数据集（定义 Schema：字段名、类型、描述）
- 数据导入：Excel（.xlsx）、JSON、Markdown、文本
- AI 辅助导入：自动解析非结构化数据
- 记录 CRUD（最多 10,000 条/数据集）
- 数据去重（基于 content hash）

**存储**：`data_center_datasets` + `data_center_records` 表

#### 📡 数据采集器（`/collector`）

定时从外部数据源采集数据，AI 解析后写入数据中心或笔记库。

**功能**：
- 定义采集任务（URL/来源 + Cron 表达式）
- 动态调度（`CollectorScheduler`）
- AI 解析采集到的原始数据
- 采集日志（最多 100 条）与结果（最多 50 条）

#### 💻 编程 AI（`/coding`）

独立飞书机器人 + Pi CLI 代码自动排查。

**功能**：
- 独立飞书机器人接收代码问题
- Pi CLI 自动执行代码排查
- 结果推回飞书

#### 🧠 AI 引导（`/ai-guide`）

当前 KB 的 AGENTS.md + AI 记忆编辑。

**功能**：
- 编辑 AGENTS.md（AI 行为规则）
- 管理 AI 记忆文件（`AI/记忆/` 目录）

#### ⚙️ 配置管理（`/config`）

全局配置 LLM / 知识库 / 系统设置。

**配置项**：
- LLM API Key、Base URL、Model
- 知识库管理（增删改查）
- 飞书集成配置
- 嵌入模型配置

#### 📜 操作日志（`/log`）

查看系统操作记录。

## 四、数据存储设计

### 4.1 核心原则

**除了 SQLite 元数据，所有业务数据都在用户的笔记库里。**

### 4.2 存储分工

| 数据类型 | 存储位置 | 说明 |
|---------|---------|------|
| **业务数据** | 各知识库的 notesDir | JSON / Markdown，全部在本地文件系统 |
| **聊天记录** | `memory.db`（SQLite） | `sessions` + `messages` 表（含 `kb_id`） |
| **对话向量** | `memory.db`（SQLite） | `turn_embeddings` 表，按 `kb_id` 隔离 |
| **知识库元数据** | `memory.db`（SQLite） | `knowledge_bases` 表（名称 + 路径 + 菜单配置） |
| **模块定义** | `memory.db`（SQLite） | `modules` 表（含 `kb_id`） |
| **数据集** | `memory.db`（SQLite） | `data_center_datasets` + `data_center_records` 表 |
| **任务/提醒** | `memory.db`（SQLite） | `tasks` + `reminders` 表（含 `kb_id`） |
| **配置信息** | `config.json`（配置目录） | LLM 凭据、飞书配置、全局设置 |
| **操作日志** | `assistant_log.json`（配置目录） | 操作时间、类型、结果 |

### 4.3 双写模式

任务和提醒采用 SQLite 为主存储 + 笔记库 JSON 双写模式：
- 创建/编辑时：先写 SQLite，再同步写入笔记库 JSON
- 启动时：如果 SQLite 为空，从笔记库 JSON 迁移数据

### 5.3 飞书集成场景

**双向集成**：
- **出站推送**：日报、提醒通过 Webhook 推送到飞书群
- **双向对话**：通过 WebSocket 长连接实时接收飞书消息，AI 处理后自动回复

**典型流程**：
1. 用户在飞书群 @AI 助理
2. `FeishuLongConnectionService` 接收消息
3. 去重检查（防止重复处理）
4. `NoteAssistantService` 处理消息（含工具编排）
5. 结果推回飞书群

### 5.4 编程 AI 场景

**独立飞书机器人**：
1. 开发者在飞书群 @编程 AI
2. 接收代码问题描述
3. Pi CLI 自动执行代码排查（读取项目文件、分析问题）
4. 结果推回飞书群

## 六、部署与运行

### 6.1 环境要求

| 组件 | 要求 | 说明 |
|------|------|------|
| JDK | 17+ | |
| Maven | 3.8+ | 仅编译需要 |
| LLM API Key | 必填 | DeepSeek / 商汤 / 智谱等兼容 OpenAI 格式的 API |
| Ollama | 可选 | 本地语义检索加速（nomic-embed-text），不装也能用 |

### 6.2 启动

```bash
# 1. 编译
mvn package -q

# 2. 启动（一条命令）
java -jar target/ai-assistant-1.0.0.jar

# 3. 打开浏览器
# http://localhost:6790
# 进入 /config 配置 LLM API Key + 笔记库根目录即可使用
```

**零外部服务**：不需要启动任何外部进程，`java -jar` 一步启动。

### 6.3 配置

**环境变量**（优先级最高）：
```bash
export LLM_API_KEY=sk-xxxxx
export LLM_BASE_URL=https://api.deepseek.com
export LLM_MODEL=deepseek-chat
```

**Web 配置页**：`/config` 页面修改 Base URL 和 Model 即可切换厂商，无需重启。

**支持的 LLM 提供商**：

| 提供商 | Base URL | 示例模型 |
|--------|----------|---------|
| DeepSeek | `https://api.deepseek.com` | `deepseek-chat` |
| 商汤 SenseNova | `https://token.sensenova.cn/v1` | `SenseChat` |
| 智谱 GLM | `https://open.bigmodel.cn/api/paas/v4` | `glm-4` |
| 本地 Ollama | `http://127.0.0.1:11434/v1` | `qwen2.5` |

### 6.4 数据隐私

- 所有业务数据保存在**用户自己的笔记库**中，绝不上传云端
- LLM API 调用只发送当前对话内容，不传输笔记库原始数据
- Ollama 语义检索完全本地运行，向量不出本机
- 聊天记录（SQLite）仅作为操作日志，不混入知识库

## 九、设计亮点

### 9.1 ChatClient 懒初始化 + 配置热更新

`LlmService` 和 `NoteAssistantService` 均使用双重检查锁懒初始化 ChatClient。当 `/config` 页面修改 LLM 配置后，下次请求会自动重建客户端，无需重启。

### 9.2 ThreadLocal 隔离

`NoteTools` 通过 `ThreadLocal<Long> CURRENT_KB_ID` 隔离不同知识库的文件操作。每次 AI 对话开始时设置，结束时清理，确保工具操作作用于正确的知识库。

### 9.3 工具自注册机制

`ToolRegistry` 在启动时扫描所有 `@Tool` 注解方法，统一管理。新增工具只需：
1. 创建 `@Component` 类
2. 添加 `@Tool` 注解方法
3. 注入到 `ToolRegistry`

无需修改任何 Service 代码。

### 9.4 语义检索降级

Ollama 不可用时，自动降级为简单时间窗口（最近 N 轮对话），确保基本功能不受影响。

### 9.5 数据双写 + 迁移

任务/提醒采用 SQLite 为主 + 笔记库 JSON 双写模式。启动时自动从 JSON 迁移到 SQLite，保证数据一致性。
