# 菜单重构设计文档

## 一、设计目标

围绕「KB 即对话」核心思想，将对话提升为 KB 内一等公民，统一 KB 内导航体验，消除当前布局的 9 个结构性问题。

**核心原则**：
1. 对话是 KB 内的首要 Tab，进入 KB 默认即对话
2. 所有 KB 相关功能（笔记/数据/索引/计划/AI 指南）都在 KB 内 Tab 中
3. 全局栏极简化，只保留跨 KB 或系统级功能
4. KB 切换器贯穿所有 KB 子页面，不再只有对话页能切

---

## 二、新菜单结构

### 顶部全局栏（极简，5 项）

```
🏠 首页 (/)                  — 工作台，KB 卡片入口
🧰 AI 能力 (/tools)          — 下拉：通用识别 / 识题 / 试卷识别
⚙️ 配置 (/config)            — 系统配置（LLM/Ollama/飞书）
📖 日志 (/log)
❓ 帮助 (/help)
```

**移除的全局项**：
- 💬 对话 → 移入 KB 内 Tab（核心改动）
- 📋 计划 → 移入 KB 内 Tab
- 📚 笔记库下拉 → 改为左侧栏 KB 列表
- 📥 数据导入 / 🔧 数据采集 / 📋 数据加工 → 移入 KB 内「数据」Tab 的子操作
- 💻 编程AI → 保留为全局条件项（开发者向，不绑定 KB）

### KB 内布局（左侧栏 + 右侧 Tab）

```
┌─────────────────┬──────────────────────────────────────────┐
│  KB 列表（左侧栏）│  KB 内 Tab（右侧主区域）                  │
│                 │                                          │
│  📂 工作笔记      │  💬 对话  📊 概览  📂 笔记  📈 数据        │
│  📂 客户管理      │  🔍 索引  📋 计划  🤖 AI指南              │
│  📂 学习笔记      │                                          │
│  📂 ...         │  ┌──────────────────────────────────────┐│
│                 │  │                                      ││
│  ＋ 新建 KB      │  │  当前 Tab 内容                        ││
│                 │  │                                      ││
└─────────────────┘  └──────────────────────────────────────┘│
```

### KB 内 Tab（7 个，按使用频率排序）

| 顺序 | Tab | 路由 | 说明 |
|------|-----|------|------|
| 1 | 💬 对话 | `/kb/{id}/chat` | **新增**。默认 Tab，进入 KB 即对话 |
| 2 | 📊 概览 | `/kb/{id}` | AI 综合日报 |
| 3 | 📂 笔记 | `/kb/{id}/notes` | 文件浏览 + 查看 |
| 4 | 📈 数据 | `/kb/{id}/data` | 数据概览 + 导入/采集/加工子操作 |
| 5 | 🔍 索引 | `/kb/{id}/index` | 语义索引管理 |
| 6 | 📋 计划 | `/kb/{id}/planner` | **新增**。任务 + 提醒（KB 范围） |
| 7 | 🤖 AI指南 | `/kb/{id}/guide` | **改名**。原 `/kb/{id}/config`，编辑 AGENTS.md + AI 记忆 |

**默认落地页**：访问 `/kb/{id}` 时，301 重定向到 `/kb/{id}/chat`（对话优先）。

## 三、路由映射表（旧 → 新）

### 新增路由

| 新路由 | Controller | 模板 | 说明 |
|--------|-----------|------|------|
| `GET /kb/{id}/chat` | `KnowledgeBaseController` | `kb_chat.html`（新） | KB 内对话页 |
| `GET /kb/{id}/planner` | `KnowledgeBaseController` | `kb_planner.html`（新） | KB 内计划页 |

### 改名路由

| 旧路由 | 新路由 | 说明 |
|--------|--------|------|
| `/kb/{id}/config` | `/kb/{id}/guide` | 「配置」→「AI指南」，消除语义冲突 |

### 废弃路由（保留 301 重定向兼容）

| 旧路由 | 重定向到 | 说明 |
|--------|---------|------|
| `/chat?kbId={id}` | `/kb/{id}/chat` | 对话页移入 KB 内 |
| `/planner` | `/`（首页） | 全局计划废弃，改为 KB 内 |
| `/kb/{id}/config` | `/kb/{id}/guide` | 改名 |
| `/kb` | `/` | 已废弃，保持现状 |
| `/ai-guide` | 删除 | 未在导航中，与 `/kb/{id}/guide` 重复 |
| `/data-editor` | 评估保留 | 未在导航中，但可能被其他页面 JS 调用 |

### 数据工具路由调整

| 旧路由 | 新位置 | 说明 |
|--------|--------|------|
| `/data/import` | `/kb/{id}/data/import` | 移入 KB 内数据 Tab 子页 |
| `/data/collector` | `/kb/{id}/data/collector` | 同上 |
| `/data/processing` | `/kb/{id}/data/processing` | 同上 |

**注意**：数据工具移入 KB 内后，必须绑定 kbId，操作当前 KB 的数据目录。

### 保留不变的路由

| 路由 | 说明 |
|------|------|
| `/` | 首页（工作台） |
| `/tools` | AI 能力入口（图像识别类） |
| `/image/general` `/image/solve` `/image/exam` | AI 识别类工具，保留全局 |
| `/config` `/log` `/help` | 系统级页面 |
| `/coding` | 编程 AI（条件显示） |
| `/kb/{id}/notes` `/kb/{id}/notes/view` | 笔记浏览/查看 |
| `/kb/{id}/index` | 索引管理 |
| `/kb/{id}/data` `/kb/{id}/data/detail` | 数据概览/详情 |

## 六、迁移步骤（建议分 4 个 PR）

### PR 1：KB 内对话 Tab（核心，最小可行改造）

**验证**：进入 KB 即对话，KB 切换时对话历史跟随切换。

### PR 3：计划并入 KB + 数据工具移入 KB
1. 新建 `kb_planner.html`，从 `planner.html` 迁移

### PR 4：清理废弃路由 + 收尾
1. 删除 `/ai-guide` 路由
2. 删除 `kb_list.html` 模板

## 七、兼容性处理

| 场景 | 处理方式 |
|------|---------|
| 用户收藏了旧 URL（`/chat?kbId=3`） | 301 重定向到 `/kb/3/chat` |
| 飞书 WebSocket 调用 `/chat/api/*` | API 路由不变，无需改动 |
| 前端 JS 中 `window.location.href = '/chat?kbId='` | 全局搜索替换为 `/kb/{id}/chat` |
| 浏览器历史记录 | 301 重定向保证可访问 |

## 八、预期效果

### 改造前
```
用户想对话：首页 → 顶部导航「对话」→ 选 KB → 开始对话
用户在 KB 笔记页想提问：返回顶部导航 → 「对话」→ 重新选 KB → 提问
```

### 改造后
```
用户想对话：首页 → 点 KB → 直接对话（默认 Tab）
用户在 KB 笔记页想提问：点「对话」Tab → 历史已加载 → 提问
```

**核心收益**：
- 对话路径从 3 步缩短到 2 步
- KB 内任意页面可一键切到对话，上下文不丢
- KB 切换贯穿所有子页面，不再只有对话页能切
- 菜单语义清晰，无「配置」冲突
